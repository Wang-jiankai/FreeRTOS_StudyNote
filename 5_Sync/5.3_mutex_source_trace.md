# 5.3 源码解析：xSemaphoreTake trace

## 互斥量的本质

互斥量在 FreeRTOS 内部是**队列长度为 1 的队列**：

```c
// xSemaphoreCreateMutex 内部实现
SemaphoreHandle_t xSemaphoreCreateMutex(void) {
    return xQueueCreateMutex(queueQUEUE_TYPE_MUTEX);
}

// queue.c
QueueHandle_t xQueueCreateMutex(unsigned char ucQueueType) {
    return xQueueGenericCreate((UBaseType_t)1, sizeof(int8_t), ucQueueType);
}
```

所以互斥量的 Take/Give 实际上就是队列的 Receive/Send。

## xSemaphoreTake 源码 trace

### Step 1：调用队列接收

```c
BaseType_t xSemaphoreTake(SemaphoreHandle_t xSemaphore, TickType_t xTicksToWait) {
    /* 实际上是调用 xQueueReceive */
    return xQueueReceive(xSemaphore, NULL, xTicksToWait);
}
```

**关键**：`xQueueReceive` 中，如果队列为空，任务被放入 `xTasksWaitingToReceive` 链表并阻塞。

### Step 2：互斥量的特殊处理

```c
// queue.c 中的 xQueueReceive
BaseType_t xQueueReceive(QueueHandle_t xQueue,

        vTaskPlaceOnEventList(&(xQueue->xTasksWaitingToReceive), xTicksToWait);
        // ...
    }
    // ...
}
```

## 优先级继承的实现

### 互斥量持有者记录

```c
typedef struct QueueDef {
    // ...
    void *pvMutexHolder;  // 当前持有互斥量的任务句柄
    UBaseType_t uxMutexHolderNesting;  // 递归嵌套计数
    // ...
} Queue_t;
```

### 优先级继承触发

```c
// queue.c
void vQueuePriorityInheritance(Queue_t *const pxQueue) {
    Task_t *pxHolder, *pxCurrentTask;

    if (pxQueue->pvMutexHolder != NULL) {
        pxHolder = (Task_t *)pxQueue->pvMutexHolder;
        pxCurrentTask = (Task_t *)xTaskGetCurrentTaskHandle();

        /* 如果持有者优先级 < 当前任务优先级，提升持有者 */
        if (pxHolder->uxPriority < pxCurrentTask->uxPriority) {
            vTaskPrioritySet(pxHolder, pxCurrentTask->uxPriority);
        }
    }
}
```

### 互斥量释放时恢复优先级

```c
void vQueueMutexUnblock(Queue_t *const pxQueue) {
    Task_t *pxHolder;

    /* 恢复原始优先级 */
    if (pxQueue->pvMutexHolder != NULL) {
        pxHolder = (Task_t *)pxQueue->pvMutexHolder;
        vTaskPrioritySet(pxHolder, pxHolder->uxBasePriority);  // 恢复基础优先级
    }
}
```

## 递归互斥量的实现

```c
typedef struct QueueDef {
    UBaseType_t uxMutexHolderNesting;  // 嵌套计数
    // ...
} Queue_t;
```

### Take 递归

```c
BaseType_t xQueueTakeMutexRecursive(QueueHandle_t xQueue, TickType_t xTicksToWait) {
    /* 直接看第几次获取 */
    if (pxQueue->uxMutexHolderNesting == 0) {
        /* 第一次获取，同普通 Take */
        xQueueReceive(xQueue, NULL, xTicksToWait);
    } else {
        /* 递归获取：不阻塞，只增加计数 */
        pxQueue->uxMutexHolderNesting++;
    }
    return pdTRUE;
}
```

### Give 递归

```c
BaseType_t xQueueGiveMutexRecursive(QueueHandle_t xQueue) {
    /* 嵌套计数减 1 */
    if (pxQueue->uxMutexHolderNesting == 0) {
        /* 没有嵌套了，真正释放 */
        xQueueGenericSend(xQueue, NULL, 0, queueSEND_TO_BACK);
    } else {
        pxQueue->uxMutexHolderNesting--;
    }
}
```

## 本节小结

- **互斥量本质** = 长度为 1 的队列
- **Take** = 从队列接收（队列空则阻塞）
- **Give** = 往队列发送
- **优先级继承**：持有者优先级被临时提升到等待者的优先级
- **递归互斥量** = 用 `uxMutexHolderNesting` 计数，支持同一任务多次获取
