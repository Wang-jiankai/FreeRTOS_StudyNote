# 5.4 优先级继承：解决优先级反转

## 什么是优先级反转

高优先级任务等待低优先级任务持有的资源，中优先级任务抢占了低优先级任务，导致高优先级任务实际执行顺序反转。

```

2. L blocks (I/O)
3. H wants same mutex, blocked, waiting for L
4. M preempts L (L is blocked!)
5. M runs (H still blocked, waiting for L)
6. M finishes
7. L continues, releases mutex
8. H finally gets mutex and runs

Label "H" (high), "M" (medium), "L" (low) with their priority numbers. Mark blocked segments. Add text showing "H 被 M 间接阻塞" in Chinese.}
```

## 优先级反转的危害

| 场景 | 问题 |
|------|------|
| 实时系统 | 响应时间不确定，违反实时性 |
| 高优先级任务 | 被无限延迟 |
| 系统不稳定 | 可能导致看门狗复位 |

## 解决方案：优先级继承

### 原理

当高优先级任务等待低优先级任务持有的互斥量时，**临时提升低优先级任务的优先级**到高优先级任务的级别，防止中优先级任务抢占。

### 效果

```
[IMAGE_PROMPT: Priority inheritance effect timeline. Same 3 tasks H(3), M(2), L(1) with mutex.
Without inheritance: H blocked by M until L finishes (bad).
With inheritance: When H wants mutex, L's priority is elevated to 3, so M cannot preempt L.
Timeline with inheritance:
1. L runs, acquires mutex
2. L blocks (I/O)
3. H wants mutex → L's priority elevated to 3
4. M cannot preempt L (L priority = 3 > M priority = 2)
5. L finishes I/O, releases mutex
6. H gets mutex and runs
M is completely blocked until L releases mutex.
Compare with previous diagram showing improvement. Label "优先级继承生效" in Chinese.}
```

## FreeRTOS 互斥量的优先级继承

### 触发时机

```c
// 任务获取互斥量时
xQueueReceive(xMutex, NULL, portMAX_DELAY);
//        ↑
//        vQueuePriorityInheritance() 被调用
```

### 实现

```c
void vQueuePriorityInheritance(Queue_t *const pxQueue) {
    Task_t *pxHolder = (Task_t *)pxQueue->pvMutexHolder;
    Task_t *pxCurrentTask = (Task_t *)xTaskGetCurrentTaskHandle();

    if (pxHolder == NULL) return;

    /* 如果当前任务优先级 > 持有者优先级 */
    if (pxCurrentTask->uxPriority > pxHolder->uxPriority) {
        /* 提升持有者到当前任务优先级 */
        vTaskPrioritySet(pxHolder, pxCurrentTask->uxPriority);
    }
}
```

### 恢复时机

```c
// 任务释放互斥量时
xQueueGenericSend(xMutex, NULL, 0, queueSEND_TO_BACK);
//        ↑
//        vQueueMutexUnblock() 被调用
```

### 恢复逻辑

```c
void vQueueMutexUnblock(Queue_t *const pxQueue) {
    Task_t *pxHolder = (Task_t *)pxQueue->pvMutexHolder;

    if (pxHolder == NULL) return;

    /* 恢复原始优先级（uxBasePriority 保存了原始优先级） */
    vTaskPrioritySet(pxHolder, pxHolder->uxBasePriority);
}
```

## 递归互斥量的优先级继承

递归互斥量需要等**所有嵌套都释放**才恢复优先级：

```c
// Take 递归：嵌套计数增加，不触发继承
if (pxQueue->uxMutexHolderNesting > 0) {
    pxQueue->uxMutexHolderNesting++;
    return pdTRUE;
}

// Give 递归：嵌套计数归零时才真正释放，恢复优先级
if (pxQueue->uxMutexHolderNesting == 0) {
    xQueueGenericSend(xMutex, NULL, 0, queueSEND_TO_BACK);
} else {
    pxQueue->uxMutexHolderNesting--;
}
```

## 优先级继承的限制

| 限制 | 说明 |
|------|------|
| 只对互斥量生效 | 二进制信号量不支持 |
| 嵌套层数有限 | 递归互斥量嵌套层数有限 |
| 不可传递 | A 持有互斥量等 B，B 等 C，不能传递继承 |
| 不解决死锁 | 只能缓解优先级反转 |

## 注意事项

### 1. 不要在中断中使用互斥量

```c
// ❌ 错误：中断中不能使用互斥量
void UART_IRQHandler(void) {
    xSemaphoreTakeFromISR(xMutex, NULL);  // ❌ 不存在这个函数！
}
```

### 2. 优先级继承可能加剧延迟

如果持有互斥量的任务被频繁提升，中优先级任务可能长时间无法运行。

### 3. 递归互斥量慎用

```c
// ❌ 递归互斥量可能导致死锁
void task_A(void *p) {
    xSemaphoreTakeRecursive(xMutex, portMAX_DELAY);
    task_B();  // task_B 也用这个互斥量
    xSemaphoreGiveRecursive(xMutex);
}

void task_B(void *p) {
    xSemaphoreTakeRecursive(xMutex, portMAX_DELAY);  // 第2次获取
    // ...
    xSemaphoreGiveRecursive(xMutex);  // 第2次释放
}
```

## 本节小结

- **优先级反转**：高优先级等低优先级，中优先级抢了低优先级
- **优先级继承**：临时提升低优先级任务的优先级，防止中优先级抢占
- **触发**：获取互斥量时提升，释放时恢复
- **注意**：互斥量才有继承，二进制信号量没有
