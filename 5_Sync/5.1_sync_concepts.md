# 5.1 信号量、互斥量、计数信号量

## 为什么需要同步原语

多任务系统中，任务之间需要协调：
- 访问共享资源（不能同时读写）
- 等待某个事件发生
- 控制任务执行顺序

队列本身可以传递数据，但**不解决**：
- 两个任务同时访问 LCD
- 任务等待某个标志位

---

## 信号量（Semaphore）

### 概念

信号量是一个**计数器**，表示可用资源数量。

| 操作 | 效果 |
|------|------|
| Take（获取） | 计数器 -1，如果 ≥ 0 获取成功 |
| Give（释放） | 计数器 +1 |

```
[IMAGE_PROMPT: Semaphore counter diagram. Show a counter box with value starting at 3. Two rows:
Row 1: "Take" operations: Take1 (3→2 success), Take2 (2→1 success), Take3 (1→0 success), Take4 (0→-1 FAIL/blocked)
Row 2: "Give" operations: Give1 (-1→0 unblock Take4), Give2 (0→1), Give3 (1→2)
Label "计数器" and "可用资源数". Use Chinese labels throughout.]
```

### 类型

| 类型 | 初始值 | 说明 |
|------|--------|------|
| **二进制信号量** | 1 或 0 | 等价于互斥量，但**不支持优先级继承** |
| **计数信号量** | N（任意） | 表示 N 个资源 |
| **互斥量** | 1 | 专门用于互斥，**支持优先级继承** |

---

## 二进制信号量

### 用途

- 任务同步（事件通知）
- 中断与任务同步

### API

```c
// 创建
SemaphoreHandle_t xSemaphore = xSemaphoreCreateBinary();

// 中断中 Give（发送）
xSemaphoreGiveFromISR(xSemaphore, &xHigherPriorityTaskWoken);

// 任务中 Take（获取）
xSemaphoreTake(xSemaphore, portMAX_DELAY);
```

### 典型用法

```c
// 中断 触发 任务
void UART_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xSemaphoreGiveFromISR(xBinarySemaphore, &xHigherPriorityTaskWoken);
    if (xHigherPriorityTaskWoken == pdTRUE) {
        portYIELD_FROM_ISR();
    }
}

void uart_task(void *p) {
    while (1) {
        xSemaphoreTake(xBinarySemaphore, portMAX_DELAY);  // 阻塞等中断
        // 处理数据
    }
}
```

## 互斥量（Mutex）

### 概念

互斥量是用于**保护共享资源**的锁。

### 与二进制信号量的区别

| 特性 | 二进制信号量 | 互斥量 |
|------|-------------|--------|
| 用途 | 同步 | 互斥 |
| 优先级继承 | ❌ | ✅ |
| 递归获取 | ❌ | ✅（递归互斥量） |
| 在中断中使用 | ✅ | ❌ |

### 优先级继承（Priority Inheritance）

解决**优先级反转**问题：

```
[IMAGE_PROMPT: Priority inheritance mechanism. Timeline showing:
- Time 1-2: Task L (low) holds mutex, then blocks on I/O
- Time 2-3: Task H (high) tries to take mutex, blocked
- Time 2.5: Task M (medium) preempts Task L and runs (wrong! H is waiting for L)
- Time 3: M finishes, L finishes I/O, releases mutex
- Time 3.5: H finally gets mutex
With priority inheritance:
- When H blocks on mutex held by L, L's priority is elevated to H's priority
- This prevents M from preempting L
Label "普通互斥量" and "带优先级继承的互斥量" comparing both scenarios. Use Chinese labels.}
```

### API

```c
// 创建（普通互斥量）
SemaphoreHandle_t xMutex = xSemaphoreCreateMutex();

// 创建（递归互斥量，可重复获取）
SemaphoreHandle_t xRecursiveMutex = xSemaphoreCreateRecursiveMutex();

// 获取（普通互斥量）
xSemaphoreTake(xMutex, portMAX_DELAY);

// 递归互斥量获取
xSemaphoreTakeRecursive(xRecursiveMutex, portMAX_DELAY);

// 释放（普通互斥量）
xSemaphoreGive(xMutex);

// 递归互斥量释放
xSemaphoreGiveRecursive(xRecursiveMutex);
```

### 正确用法

```c
// ✅ 标准互斥量使用
void access_shared_resource(void *p) {
    xSemaphoreTake(xMutex, portMAX_DELAY);  // 获取锁
    // 访问共享资源
    xSemaphoreGive(xMutex);                  // 释放锁
}

// ❌ 错误：忘记释放
void bad_access(void *p) {
    xSemaphoreTake(xMutex, portMAX_DELAY);
    if (error_condition) {
        return;  // 忘了 Give！死锁
    }
    xSemaphoreGive(xMutex);
}
```

## 计数信号量

### 概念

计数器可以 > 1，表示多个可用资源。

### 典型场景

```c
// 场景：3 个相同的资源（如串口缓冲区）
SemaphoreHandle_t xResourceSemaphore = xSemaphoreCreateCounting(3, 3);
//                                         ^^^^^  ^^^^^
//                                         最大值  初始值

// 获取一个资源
xSemaphoreTake(xResourceSemaphore, portMAX_DELAY);
// 使用资源...
xSemaphoreGive(xResourceSemaphore);  // 用完释放
```

## 死锁与预防

### 死锁四条件

| 条件 | 说明 |
|------|------|
| 互斥 | 资源只能被一个任务持有 |
| 持有并等待 | 持有资源同时等待更多资源 |
| 不可抢占 | 资源不能被强制夺走 |
| 循环等待 | 任务形成环形等待 |

### 预防方法

```c
// ✅ 方法 1：固定顺序获取锁
// 任务 A 和 B 都必须先拿 mutex1 再拿 mutex2
void task_A(void *p) {
    xSemaphoreTake(mutex1, portMAX_DELAY);
    xSemaphoreTake(mutex2, portMAX_DELAY);
    // ...
    xSemaphoreGive(mutex2);
    xSemaphoreGive(mutex1);
}

void task_B(void *p) {
    xSemaphoreTake(mutex1, portMAX_DELAY);  // 顺序一致，不会死锁
    xSemaphoreTake(mutex2, portMAX_DELAY);
    // ...
}

// ✅ 方法 2：加超时
if (xSemaphoreTake(mutex1, pdMS_TO_TICKS(100)) == pdTRUE) {
    if (xSemaphoreTake(mutex2, pdMS_TO_TICKS(100)) == pdTRUE) {
        // ...
        xSemaphoreGive(mutex2);
    }
    xSemaphoreGive(mutex1);
} else {
    // 超时，放弃或重试
}
```

## 本节小结

- **二进制信号量**：事件同步（中断→任务）
- **互斥量**：保护共享资源，支持优先级继承
- **计数信号量**：资源池（多个相同资源）
- **优先级继承**：解决互斥量导致的优先级反转
- **死锁预防**：固定顺序获取锁、加超时
