# 5.5 踩坑实录：死锁 / 忘记 Give

## 坑 1：忘记 Give（最常见）

### 现象

互斥量永久阻塞，其他任务永远拿不到。

### 代码

```c
void bad_task(void *p) {
    xSemaphoreTake(xMutex, portMAX_DELAY);

    if (error_condition) {
        return;  // ❌ 提前返回，没 Give！
    }

    // 正常流程
    xSemaphoreGive(xMutex);  // 不会执行到
}
```

### 解决

```c
// ✅ 方法 1：所有出口都 Give
void good_task(void *p) {
    xSemaphoreTake(xMutex, portMAX_DELAY);

    if (error_condition) {
        xSemaphoreGive(xMutex);  // ✅ 清理
        vTaskDelete(NULL);
    }

    // 正常流程
    xSemaphoreGive(xMutex);  // ✅
}

// ✅ 方法 2：用 goto 统一出口
void better_task(void *p) {
    xSemaphoreTake(xMutex, portMAX_DELAY);

    if (error_condition) {
        goto cleanup;
    }

    // 正常流程

cleanup:
    xSemaphoreGive(xMutex);
}
```

## 坑 2：死锁

### 现象

两个任务互相等待对方持有的互斥量，系统卡死。

### 代码

```c
// ❌ 死锁
void task_A(void *p) {
    xSemaphoreTake(mutex1, portMAX_DELAY);
    vTaskDelay(pdMS_TO_TICKS(10));  // 假装做点事
    xSemaphoreTake(mutex2, portMAX_DELAY);  // 等 mutex2

    // ...
    xSemaphoreGive(mutex2);
    xSemaphoreGive(mutex1);
}

void task_B(void *p) {
    xSemaphoreTake(mutex2, portMAX_DELAY);
    vTaskDelay(pdMS_TO_TICKS(10));
    xSemaphoreTake(mutex1, portMAX_DELAY);  // 等 mutex1

    // ...
    xSemaphoreGive(mutex1);
    xSemaphoreGive(mutex2);
}
```

```
[IMAGE_PROMPT: Deadlock diagram. Two tasks and two mutexes. Task A holds Mutex1, waits for Mutex2. Task B holds Mutex2, waits for Mutex1. Circular arrows showing mutual wait. Label "死锁" and "A 等 B，B 等 A，永远无法继续" in Chinese.}
```

### 解决

```c
// ✅ 固定顺序获取
void task_A(void *p) {
    xSemaphoreTake(mutex1, portMAX_DELAY);
    xSemaphoreTake(mutex2, portMAX_DELAY);  // 顺序和 B 一致

    // ...
    xSemaphoreGive(mutex2);
    xSemaphoreGive(mutex1);
}

void task_B(void *p) {
    xSemaphoreTake(mutex1, portMAX_DELAY);  // ✅ 顺序和 A 一致
    xSemaphoreTake(mutex2, portMAX_DELAY);

    // ...
    xSemaphoreGive(mutex2);
    xSemaphoreGive(mutex1);
}
```

## 坑 3：中断中误用互斥量

### 现象

编译报错或系统行为异常。

### 代码

```c
// ❌ 互斥量不能在中断中使用
void UART_IRQHandler(void) {
    xSemaphoreTake(xMutex, 0);  // ❌ 不支持！
}
```

### 解决

```c
// ✅ 中断同步用二进制信号量
void UART_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xSemaphoreGiveFromISR(xBinarySem, &xHigherPriorityTaskWoken);
    if (xHigherPriorityTaskWoken == pdTRUE) {
        portYIELD_FROM_ISR();
    }
}
```

## 坑 4：递归互斥量忘记嵌套次数

### 现象

递归获取多次，但只释放一次，互斥量永久持有。

### 代码

```c
// ❌ 嵌套次数不对
void outer_func(void *p) {
    xSemaphoreTakeRecursive(xMutex, portMAX_DELAY);
    inner_func();
    xSemaphoreGiveRecursive(xMutex);  // ❌ 只释放了1次，但获取了2次
}

void inner_func(void *p) {
    xSemaphoreTakeRecursive(xMutex, portMAX_DELAY);
    // ...
    // 没给！
}
```

### 解决

```c
// ✅ 确保配对
void outer_func(void *p) {
    xSemaphoreTakeRecursive(xMutex, portMAX_DELAY);
    inner_func();
    xSemaphoreGiveRecursive(xMutex);  // ✅
}

void inner_func(void *p) {
    xSemaphoreTakeRecursive(xMutex, portMAX_DELAY);
    // ...
    xSemaphoreGiveRecursive(xMutex);  // ✅
}
```

## 坑 5：多次 Give 同一信号量

### 现象

信号量计数超出最大值，导致不可预期的行为。

### 代码

```c
// ❌ 多次 Give
void bad_task(void *p) {
    xSemaphoreGive(xMutex);  // 第1次
    xSemaphoreGive(xMutex);  // ❌ 第2次，计数变成2
}
```

### 解决

```c
// ✅ 确保 Give 和 Take 配对
void correct_task(void *p) {
    xSemaphoreTake(xMutex, portMAX_DELAY);
    // ...
    xSemaphoreGive(xMutex);  // ✅ 只在 Take 之后 Give
}
```

## 坑 6：删除持有信号的互斥量

### 现象

删除后，其他等待任务永久阻塞。

### 代码

```c
// ❌
void task_A(void *p) {
    xSemaphoreTake(xMutex, portMAX_DELAY);
    // 没释放就删除
    vTaskDelete(NULL);
}

void task_B(void *p) {
    while (1) {
        xSemaphoreTake(xMutex, portMAX_DELAY);  // 永远等不到
    }
}
```

### 解决

```c
// ✅ 删除前确保已释放
void safe_delete_mutex(void) {
    // 确保当前任务不持有互斥量
    TaskHandle_t holder = xSemaphoreGetMutexHolder(xMutex);
    if (holder == xTaskGetCurrentTaskHandle()) {
        xSemaphoreGive(xMutex);
    }
    vSemaphoreDelete(xMutex);
}
```

## 同步调试清单

```
□ 所有 xSemaphoreTake 都有对应的 xSemaphoreGive？
□ 所有 return/break/goto 出口都释放了信号量？
□ 多个互斥量是否按固定顺序获取？
□ 中断中是否使用了正确的 FromISR API？
□ 递归互斥量的 Take/Give 次数是否配对？
□ 是否删除了持有信号的互斥量？
□ 互斥量是否在持有期间调用了阻塞操作？
□ 是否用 xSemaphoreGetMutexHolder 检查了自己是否持有？
```

## 本节小结

- **忘记 Give**：所有出口路径都要释放，用 goto 统一出口
- **死锁**：固定顺序获取多个互斥量
- **中断误用**：互斥量不能用于中断，用二进制信号量代替
- **递归互斥量**：Take 和 Give 必须严格配对
- **多次 Give**：确保 Give 只在成功 Take 之后
- **删除持有者**：删除前先检查是否被持有
