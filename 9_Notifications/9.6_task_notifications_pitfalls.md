# 9.6 踩坑实录：任务通知

## 坑 1：覆盖模式丢数据

**现象**：多个 ISR 快速发送通知，数据丢失。

```c
// ISR 1 发送
xTaskNotifyFromISR(xTask, value1, eSetValueWithOverwrite, NULL);

// ISR 2 几乎同时发送
xTaskNotifyFromISR(xTask, value2, eSetValueWithOverwrite, NULL);
```

**原因**：`eSetValueWithOverwrite` 直接覆盖，如果两次通知间隔很短，value1 会丢失。

**解决**：

```c
// ✅ 用 eSetValueWithoutOverwrite（未被读取才写）
xTaskNotifyFromISR(xTask, value, eSetValueWithoutOverwrite, NULL);

// ✅ 或者用递增模式，接收端用 ulTaskNotifyTake
xTaskNotifyFromISR(xTask, 1, eIncrement, NULL);
```

---

## 坑 2：通知值溢出

**现象**：通知值一直递增，最终溢出。

```c
void high_freq_task(void *p) {
    while (1) {
        xTaskNotifyGive(xTask);  // 每次循环都递增
    }
}
```

**原因**：`uint32_t` 通知值无界限，频繁递增会溢出（变成 0 再递增）。

**解决**：

```c
// ✅ 用 eNoAction + 接收端自行处理
// 或者用带超时的 ulTaskNotifyTake 定期清零
uint32_t count = ulTaskNotifyTake(pdTRUE, pdMS_TO_TICKS(1000));
// 每秒最多累积的计数在安全范围内
```

---

## 坑 3：通知值与标志混淆

**现象**：用 `eSetBits` 设置了位，但接收端用 `ulTaskNotifyTake`（期望数值），结果不对。

```c
// 发送端
xTaskNotify(xTask, BIT_0 | BIT_1, eSetBits);  // 设置位

// 接收端
uint32_t val = ulTaskNotifyTake(pdTRUE, portMAX_DELAY);  // ❌ 得到的是位掩码，不是期望的数值
if (val & BIT_0) { ... }  // 用 ulTaskNotifyTake 无法区分"收到通知"和"收到的是位掩码"
```

**原因**：`ulTaskNotifyTake` 返回的是递减前的通知值，不是通知状态。`eSetBits` 的结果应该用 `xTaskNotifyWait` 接收。

**解决**：

```c
// ✅ 用 eSetBits 时，接收端用 xTaskNotifyWait
uint32_t val;
xTaskNotifyWait(0, 0, &val, portMAX_DELAY);
if (val & BIT_0) { ... }  // 正确判断位
```

## 坑 4：在中断中用错 API

**现象**：编译报错或者没有效果。

```c
void IRQ_Handler(void) {
    // ❌ xTaskNotifyGive 不是 FromISR 版本！
    xTaskNotifyGive(xTask);

    // ✅ 正确：用 FromISR 版本
    xTaskNotifyGiveFromISR(xTask, &xHigherPriorityTaskWoken);
}
```

**解决**：ISR 中必须用 `*FromISR` 版本。

---

## 坑 5：任务未使能任务通知

**现象**：`configUSE_TASK_NOTIFICATIONS` 没定义为 1，API 不生效。

```c
// FreeRTOSConfig.h
#define configUSE_TASK_NOTIFICATIONS  0  // ❌ 关闭了！

// 然后调用
xTaskNotifyGive(xTask);  // 不生效或崩溃
```

**解决**：

```c
// FreeRTOSConfig.h
#define configUSE_TASK_NOTIFICATIONS  1  // ✅ 开启
```

---

## 坑 6：通知对象被删除后仍通知

**现象**：任务被删除后，ISR 还在通知它，导致问题。

```c
void delete_task(void *p) {
    vTaskDelete(xTask);  // 删除了 xTask
    xTask = NULL;
}

void IRQ_Handler(void) {
    if (xTask != NULL) {
        xTaskNotifyGiveFromISR(xTask, NULL);  // ✅ 先检查
    }
}
```

---

## 本节小结

| 坑 | 原因 | 解决 |
|----|------|------|
| 覆盖丢数据 | `eSetValueWithOverwrite` 覆盖 | `eSetValueWithoutOverwrite` 或递增模式 |
| 通知值溢出 | 无限递增 | 定期清零或用位掩码模式 |
| 位与数值混淆 | `eSetBits` 应配 `xTaskNotifyWait` | 按通知方式选择接收 API |
| FromISR 用错 | 用了非 FromISR 版本 | ISR 必须用 `*FromISR` 版本 |
| 功能未开启 | `configUSE_TASK_NOTIFICATIONS = 0` | 改为 1 |
| 通知已删除任务 | 任务句柄失效 | 通知前检查句柄是否为 NULL |
