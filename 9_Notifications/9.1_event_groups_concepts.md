# 9.1 概念：事件标志组（Event Groups）

## 什么是事件标志组

事件标志组是一组**可以独立置位/清除的位**，用于任务间同步。

```

```

**典型场景**：任务 A 等待多个事件（温度传感器 + 湿度传感器 + 按钮）全部就绪后，再执行下一步。

---

## 核心概念

### 位与事件

| 位 | 含义（举例） |
|----|-------------|
| bit0 | 温度数据就绪 |
| bit1 | 湿度数据就绪 |
| bit2 | 按钮按下 |
| ... | ... |

每个位代表一个事件，位数由 `EventBits_t`（即 `TickType_t`）决定：
- 32 位架构：`EventBits_t` = `uint32_t`（32 个事件位）
- 16 位架构：`EventBits_t` = `uint16_t`（16 个事件位）

---

## 等待模式

### 等待任意位（OR）

任务等待任意一个事件位置位：

```c
EventBits_t xEventGroupWaitBits(
    xEventGroup,
    EventBits_t uxBitsToWaitFor,   /* 等待的位，如 bit0 | bit1 */
    pdFALSE,                         /* xClearOnExit = 不自动清除 */
    pdFALSE,                         /* xWaitForAllBits = OR（任意一个）*/
    portMAX_DELAY                    /* 无限等待 */
);
```

### 等待所有位（AND）

任务等待所有事件位全部置位：

```c
EventBits_t xEventGroupWaitBits(
    xEventGroup,
    EventBits_t uxBitsToWaitFor,   /* bit0 | bit1 */
    pdFALSE,
    pdTRUE,                         /* xWaitForAllBits = AND（全部）*/
    portMAX_DELAY
);
```

---

## 与其他同步方式的区别

| 特性 | 事件标志组 | 二进制信号量 | 队列 |
|------|-----------|-------------|------|
| 传递数据 | ❌ | ❌ | ✅ |
| 多个任务同步 | ✅ | ✅ | ✅ |
| 等待多个条件 | ✅ | ❌ | ❌ |
| 广播（多个任务同时就绪）| ✅ | ❌ | ✅ |

---

## 本节小结

- **事件标志组** = `EventBits_t` 类型的多个位，每位代表一个事件
- **xEventGroupWaitBits** 支持 OR（任意位）和 AND（所有位）两种等待模式
- 适用于"多个条件任一满足"或"多个条件全部满足"的同步场景
- 本身不传递数据，只传递"事件已发生"的信息
