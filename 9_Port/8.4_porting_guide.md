## FreeRTOSConfig.h 配置

### 必须配置的参数

```c
// FreeRTOSConfig.h

// ========== 系统时钟 ==========
#define configCPU_CLOCK_HZ              (72000000)  // CPU 主频
#define configSYSTICK_CLOCK_HZ          (configCPU_CLOCK_HZ / 8)  // 或相同

// ========== Tick ==========
#define configTICK_RATE_HZ              (1000)      // 1ms Tick

// ========== 中断 ==========
#define configMAX_SYSCALL_INTERRUPT_PRIORITY    (5)   // 可调
#define configPRIO_BITS                       (4)   // 优先级位数

// ========== 内存 ==========
#define configTOTAL_HEAP_SIZE            (4096 * 1024)  // heap 大小

// ========== 任务 ==========
#define configMAX_PRIORITIES             (5)            // 优先级数量
#define configMINIMAL_STACK_SIZE         (128)          // 最小栈大小

// ========== 可选功能 ==========
#define configUSE_PREEMPTION             1
#define configUSE_TIME_SLICING           1
#define configUSE_IDLE_HOOK              1
#define configUSE_TICK_HOOK              0
#define configUSE_MALLOC_FAILED_HOOK    1
```

### 与芯片相关的配置

```c
// 如果有 FPU：用 ARM_CM4F 或 ARM_CM7 移植层（而非 ARM_CM3）

// 如果需要 tickless
```

## port.c 修改

### vPortSetupTimerInterrupt

根据芯片的定时器配置 SysTick：

```c:port.c
// STM32F4 示例
void vPortSetupTimerInterrupt(void) {
    // 配置 SysTick
    SysTick->LOAD = (configCPU_CLOCK_HZ / configTICK_RATE_HZ) - 1;
    SysTick->VAL = 0;
    SysTick->CTRL = SysTick_CTRL_TICKINT_Msk |
                    SysTick_CTRL_CLKSOURCE_Msk |
                    SysTick_CTRL_ENABLE_Msk;
}
```

### xPortStartScheduler

```c:port.c
BaseType_t xPortStartScheduler(void) {
    /* 配置 PendSV 优先级 */
    NVIC_SetPriority(PendSV_IRQn, 0xFF);

    /* 配置 SysTick */
    vPortSetupTimerInterrupt();

    /* 启动第一个任务 */
    vPortStartFirstTask();

    return 0;
}
```

### vPortStartFirstTask

```c:port.c
void vPortStartFirstTask(void) {
    __asm volatile (
        " LDR r0, =0xE000ED04  \n"  // Vector table offset
        " LDR r0, [r0]         \n"
        " MSR msp, r0           \n"  // 设置主栈指针
        " CPSIE i               \n"  // 开中断
        " SVC #0                \n"  // 触发 SVC
    );
}
```

## 链接脚本修改

```ld
/* GCC 链接脚本示例 */
MEMORY
{
    FLASH (rx)  : ORIGIN = 0x08000000, LENGTH = 256K
    RAM (rwx)   : ORIGIN = 0x20000000, LENGTH = 64K
}

_stack_size = 0x1000;  /* 4KB 主栈 */

/* .text 段放 Flash */
.text :
{
    *(.text)
    *(.text.*)
    *(.rodata)
} > FLASH

/* .data 和 .bss 放 RAM */
.data :
{
    *(.data)
} > RAM AT > FLASH

.bss :
{
    *(.bss)
} > RAM
```

## 移植检查清单

### 编译前检查

- [ ] `configCPU_CLOCK_HZ` 与实际芯片主频一致
- [ ] `configTICK_RATE_HZ` 选择合适（1000 或 100）
- [ ] `configMAX_SYSCALL_INTERRUPT_PRIORITY` 不超过优先级位数
- [ ] 栈大小足够所有任务使用

### 运行时检查

- [ ] SysTick 中断每 1ms 触发一次
- [ ] `uxTaskGetCount` 显示任务数正确
- [ ] `vTaskDelay` 延时符合预期
- [ ] 上下文切换正常

### 调试工具

```c
// 打印当前任务名
extern TaskHandle_t xCurrentTaskHandle;
void debug_task(void) {
    if (xCurrentTaskHandle != NULL) {
        printf("Running: %s\n",
            pcTaskGetName(xCurrentTaskHandle));
    }
}
```

## 常见移植问题

### 问题 1：HardFault

**检查**：
- 栈是否足够大
- 堆内存是否足够
- 中断向量是否正确

### 问题 2：定时器不工作

**检查**：
- SysTick 时钟源配置
- `configCPU_CLOCK_HZ` 是否正确
- 中断优先级是否低于 configMAX_SYSCALL_INTERRUPT_PRIORITY

### 问题 3：内存溢出

**检查**：
- `configTOTAL_HEAP_SIZE` 是否足够
- 任务栈是否太大
- 是否有内存泄漏

## 本节小结

- 移植 FreeRTOS 主要改 5 个地方：配置、汇编、定时器、启动文件、链接脚本
- `FreeRTOSConfig.h` 是最常需要修改的文件
- `port.c` 需要适配具体芯片的中断控制器
- 移植后用清单逐项检查，确保系统正常运行
