# 8.3 Keil / IAR / GCC 差异

## 编译器差异概览

| 特性 | Keil (ARMCC) | IAR (ICCARM) | GCC (ARM GCC) |
|------|-------------|--------------|---------------|
| 汇编语法 | ARMASM | IAR ASM | GNU ASM |
| 关键字 | `__asm` | `__asm` | `asm` |
| 内存对齐 | `#pragma data_alignment` | `#pragma data_alignment` | `__attribute__((aligned))` |
| 内联汇编 | `__asm` | `__asm` | `__attribute__((naked))` |
| 启动文件 | `startup_xxx.s` | `startup_xxx.s` | `startup_xxx.c` |

---

## 汇编语法差异

### 函数声明

```c
// Keil (ARMCC)
void PendSV_Handler(void) __attribute__((naked));

// IAR
void PendSV_Handler(void) __nested;

// GCC
void PendSV_Handler(void) __attribute__((naked));
```

### 内联汇编

```c
// Keil
__asm void function(void) {
    MRS r0, PSP
    LDR r1, =pxCurrentTCB
}

// IAR
__stackless void function(void) {
    __asm(" MRS r0, PSP");
    __asm(" LDR r1, =pxCurrentTCB");
}

// GCC
void function(void) {
    __asm volatile (
        " MRS r0, PSP  \n"
        " LDR r1, =pxCurrentTCB  \n"
    );
}
```

## 启动文件差异

### Keil 启动文件结构

```assembly
; Keil startup_ARMCM4.s
                AREA    RESET, DATA, READONLY
                EXPORT  __Vectors

__Vectors       DCD     __initial_sp          ; Top of Stack
                DCD     Reset_Handler         ; Reset Handler
                DCD     PendSV_Handler        ; PendSV Handler
                ; ...
```

### IAR 启动文件结构

```assembly
; IAR startup_ARMCM4.s
                SECTION CSTACK:DATA:NOROOT(3)
                SECTION .intvec:CODE:NOROOT(2)

                PUBLIC  __vector_table
                DATA
__vector_table  DCD     sfe(CSTACK)           ; Top of Stack
                DCD     Reset_Handler          ; Reset Handler
                DCD     PendSV_Handler         ; PendSV Handler
```

### GCC 启动文件结构

```c
// GCC: 用 C 写启动代码
__attribute__((section(".isr_vector")))
void (* const g_pfnVectors[])(void) = {
    (void (*)(void))((uint32_t)_estack),
    Reset_Handler,
    PendSV_Handler,
    // ...
};
```

## 栈空间定义

### Keil

```c
// Keil: 在 startup 文件中定义
Stack_Size      EQU     0x00000400
                AREA    STACK, NOINIT, READWRITE, ALIGN=3
Stack_Mem       SPACE   Stack_Size
__initial_sp    EQU     Stack_Mem + Stack_Size
```

### IAR

```c
// IAR: 在 icf 文件中定义
define symbol __ICFEDIT_region_STARTSIZE__ = 0x400;
define region CSTACK_region = mem:[from _CSTACK_region_start to _CSTACK_region_end];
```

### GCC

```c
// GCC: 在链接脚本中定义
_stack_size = 0x400;
_start_stack = ORIGIN(RAM) + LENGTH(RAM);
```

## 中断向量表位置

### Keil

```c
// system_xxx.c
SCB->VTOR = (uint32_t)__Vectors;
```

### IAR

```c
// IAR: 在 icf 文件中配置
place at address mem:__vector_table_start { readonly section .intvec };
```

### GCC

```c
// GCC: 链接脚本
.rodata : {
    *(.isr_vector)
}
```

## 移植时的修改点

当把 FreeRTOS 从一个编译器迁移到另一个时：

| 文件 | 修改内容 |
|------|---------|
| `portmacro.h` | 数据类型、汇编语法、关键字 |
| `port.c` | 汇编函数实现 |
| `portmacro.h` | `__asm` → `asm` 等关键字 |
| 启动文件 | 栈大小、向量表格式 |
| 链接脚本 | 内存布局、栈位置 |

---

## 实战：从 IAR 迁移到 GCC

### 1. 修改 portmacro.h

```c
// IAR 版本
#define portSAVE_CONTEXT()         __asm(" PUSH {R4-R11}")

// GCC 版本
#define portSAVE_CONTEXT()         __asm volatile (\
    " PUSH {R4-R11}  \n"           \
    ::: "memory")
```

### 2. 修改 PendSV_Handler

```c
// IAR
__task void PendSV_Handler(void) {
    __asm(" PUSH {R4-R11}");
    // ...
}

// GCC
void PendSV_Handler(void) {
    __asm volatile (
        " PUSH {R4-R11}  \n"
        // ...
    );
}
```

### 3. 修改链接脚本

```ld
/* GCC 链接脚本 */
MEMORY
{
    FLASH (rx) : ORIGIN = 0x08000000, LENGTH = 256K
    RAM (rwx)  : ORIGIN = 0x20000000, LENGTH = 64K
}

_stack_size = 0x400;
```

## 编译器特定优化

### GCC 的 naked 函数

```c
// GCC: naked 函数不会生成函数 prologue/epilogue
void PendSV_Handler(void) __attribute__((naked));
```

### IAR 的 __stackless

```c
// IAR: stackless 不使用栈帧
__stackless void task(void);
```

### Keil 的 __asm

```c
// Keil: 内联汇编
__asm void function(void) {
    MOVS r0, #1
}
```

---

## 本节小结

- 三大编译器的主要差异在汇编语法和启动文件格式
- `portmacro.h` 和 `port.c` 需要根据编译器调整
- 关键是中断向量表的定义和栈空间的配置
- 移植时重点改汇编相关代码和链接脚本
