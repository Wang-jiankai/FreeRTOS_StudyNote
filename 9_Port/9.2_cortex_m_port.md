# 8.2 ARM Cortex-M 移植要点

## Cortex-M 移植核心

| 要素 | 说明 |
|------|------|
| 栈生长方向 | 向低地址生长（满递减） |
| 数据类型 | 32 位 |
| 上下文切换 | PendSV 中断 |
| 中断优先级 | 数值越小优先级越高 |
| 栈对齐 | 8 字节对齐 |

## 栈初始化

```c:port.c
StackType_t *pxPortInitialiseStack(
    StackType_t *pxTopOfStack,
    TaskFunction_t pxCode,
    void *pvParameters)
{
    /* 模拟中断栈帧 */
    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)portINITIAL_XPSR;  // xPSR = 0x01000000（Thumb bit 必须置 1）

    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)pxCode | portTHUMB_MODE_BIT;  // PC = 任务入口

    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)portTASK_RETURN_ADDRESS;  // LR = 任务返回地址

    /* R0-R12 初始化为 0 */
    for (int i = 12; i >= 0; i--) {
        pxTopOfStack--;
        *pxTopOfStack = 0;
    }

    /* R0 = 任务参数 */
    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)pvParameters;

    return pxTopOfStack;
}
```

## 上下文切换机制

### 触发方式

```c:port.c
// FreeRTOS 正确实现：设置 ICSR[28] 触发 PendSV
#define portYIELD() \
    do { \
        portNVIC_INT_CTRL_REG = portNVIC_PENDSVSET_BIT; \
        __asm volatile ("dsb" ::: "memory"); \
        __asm volatile ("isb"); \
    } while (0)

// 其中：
// #define portNVIC_INT_CTRL_REG   (*(volatile uint32_t *)0xE000ED04)  // SCB->ICSR
// #define portNVIC_PENDSVSET_BIT  (1UL << 28UL)  // ICSR 第 28 位
```

> ❌ 曾经错误写法：`#define portYIELD() __asm volatile ("svc 0")`  
> SVC 0 触发的是 SVC 中断，不是 PendSV！FreeRTOS 上下文切换是通过 PendSV 实现的。

### PendSV 优先级

```c:port.c
// 在 xPortStartScheduler 中设置
NVIC_SetPriority(PendSV_IRQn, 0xFF);  // 最低优先级
```

**为什么 PendSV 设最低优先级？** 确保所有更重要的中断处理完才进行任务切换。

## Cortex-M3/M4/M7 差异

| 特性 | M3 | M4 | M7 |
|------|----|----|-----|
| 架构 | v7-M | v7E-M | v7E-M |
| FPU | ❌ | 可选 | 可选 |
| DSP | ❌ | 可选 | 可选 |
| 移植层 | ARM_CM3 | ARM_CM4F | ARM_CM7 |

### FPU 支持

Cortex-M4/M7 的 FPU 支持不需要在 FreeRTOSConfig.h 中加特殊宏。正确做法：

**1. 选用正确的移植层**：用 `ARM_CM4F`（有 FPU）或 `ARM_CM7` 代替 `ARM_CM3`

**2. 编译器添加 FPU 标志**（以 STM32F4 为例，Keil）：
```c
// Options for Target → Target → Floating Point Hardware → Hardware FP
// 或在 ARM CC 编译选项中加：-mfpu=fpv4-sp-d16 -mfloat-abi=hard
```

**3. 确保任务栈足够大**：FPU 上下文（s0-s15, FPSCR）每个任务需要额外 34 words

## 常见移植问题

### 1. 堆栈对齐错误

```c
// ❌ 错误：堆栈地址不是 8 的倍数
void bad_task(void *p) {
    uint64_t big_var;  // 可能在栈上，但地址可能不对齐
}

// ✅ 正确：手动对齐
void good_task(void *p) {
    __attribute__((aligned(8))) uint64_t big_var;
}
```

### 2. 遗忘 FPU 保存

如果用了 FPU 但没在任务栈帧中保存，会导致数据丢失。

### 3. 中断优先级设错

PendSV 必须设为**最低优先级**（数值最大）。

## 本节小结

- Cortex-M 移植核心：PendSV 切换、8字节对齐、栈帧布局
- `pxPortInitialiseStack` 构造初始栈帧，模拟中断刚发生
- PendSV 必须设为最低优先级，确保其他中断先处理
- M4/M7 有可选 FPU，需要额外配置
