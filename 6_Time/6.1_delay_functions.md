# 6.1 vTaskDelay vs vTaskDelayUntil

## 两种延时函数对比

| 函数 | 特点 | 适用场景 |
|------|------|---------|
| `vTaskDelay()` | 相对延时，有抖动 | 非周期性任务 |
| `vTaskDelayUntil()` | 绝对延时，精确周期 | 周期性任务 |

---

## vTaskDelay（相对延时）

### 行为

延时是相对于**调用时刻**的，任务从调用时刻开始阻塞。

```

Label "vTaskDelay(10ms) 从调用时刻开始延时". Show execution and blocked segments clearly. Note that interval between executions is ~10ms but not exact due to scheduling delays.]
```

### 代码

```c
void sensor_task(void *p) {
    while (1) {
        uint32_t value = read_sensor();

        vTaskDelay(pdMS_TO_TICKS(100));  // 延时 100ms

        // 两次执行的间隔 ≈ 100ms + 调度开销
    }
}
```

## vTaskDelayUntil（绝对延时）✅ 推荐

### 行为

延时是相对于**上一次唤醒时刻**的，保证固定周期。

```c
void precise_task(void *p) {
    TickType_t xLastWakeTime = xTaskGetTickCount();  // 初始化

    while (1) {
        process();

        // 下一次唤醒时间 = 上次唤醒时间 + period
        // 任务执行时间不影响周期
        vTaskDelayUntil(&xLastWakeTime, pdMS_TO_TICKS(100));
    }
}
```

```
[IMAGE_PROMPT: vTaskDelayUntil timeline showing precise periodic execution. Same 3 cycles but now execution always starts at exact multiples of 10ms: 10ms, 20ms, 30ms. Even if a cycle is delayed, the next wake time is calculated from the last wake, not from now. Show formula: next_wake = last_wake + period. Label "vTaskDelayUntil 保证精确周期" and "抖动被消除" in Chinese.]
```

### 源码逻辑

```c
void vTaskDelayUntil(TickType_t *pxPreviousWakeTime, TickType_t xTimeIncrement) {
    TickType_t xTimeToWake;
    BaseType_t xShouldDelay = pdFALSE;

    vTaskSuspendAll();

    /* 计算下次唤醒时间 */
    xTimeToWake = *pxPreviousWakeTime + xTimeIncrement;

    /* 如果唤醒时间已过（被调度延迟），立即返回不延时 */
    if (xTimeToWake > xTaskGetTickCount()) {
        *pxPreviousWakeTime = xTimeToWake;
        xShouldDelay = pdTRUE;
    }

    xTaskResumeAll();

    if (xShouldDelay) {
        vTaskDelay(xTimeToWake - xTaskGetTickCount());
    }
}
```

---

## 实际选择

| 场景 | 用哪个 | 理由 |
|------|--------|------|
| 周期性传感器读取 | `vTaskDelayUntil` | 保证精确周期 |
| 周期性显示刷新 | `vTaskDelayUntil` | 保证帧率稳定 |
| 一次性延时 | `vTaskDelay` | 相对延时即可 |
| 等待某个事件（非周期） | `vTaskDelay` | 等待类任务 |

---

## 常见错误

### 错误：初始化放错位置

```c
// ❌ 错误：xLastWakeTime 在循环内初始化
void bad_task(void *p) {
    while (1) {
        TickType_t xLastWakeTime = xTaskGetTickCount();  // ❌ 每次循环重置
        process();
        vTaskDelayUntil(&xLastWakeTime, pdMS_TO_TICKS(100));
    }
}

// ✅ 正确：初始化在循环外
void good_task(void *p) {
    TickType_t xLastWakeTime = xTaskGetTickCount();  // ✅ 只初始化一次
    while (1) {
        process();
        vTaskDelayUntil(&xLastWakeTime, pdMS_TO_TICKS(100));
    }
}
```

---

## 本节小结

- **vTaskDelay**：相对延时，有抖动，适合非周期性任务
- **vTaskDelayUntil**：绝对延时，无抖动，保证精确周期
- **周期性任务用 vTaskDelayUntil**，初始化在循环外
- **注意**：`xLastWakeTime` 只初始化一次，不在循环内重置
