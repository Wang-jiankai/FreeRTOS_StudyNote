# 6.2 Tick 中断机制与软件定时器 daemon

## Tick 中断是什么

Tick 中断是 FreeRTOS 的**系统时钟**，由硬件定时器周期性触发（通常 1ms/次）。

```
[IMAGE_PROMPT: Tick interrupt timeline. Time axis showing tick interrupts at every 1ms (assuming configTICK_RATE_HZ = 1000). Each tick shows small interrupt pulse, followed by xTaskIncrementTick() execution. Label "SysTick 中断" and "configTICK_RATE_HZ = 1000 → 每 1ms 一次". Show that during tick execution, task is briefly preempted. Use Chinese labels.}
```

## Tick 中断的作用

| 作用 | 说明 |
|------|------|
| 延时管理 | 检查延时队列，唤醒到期的任务 |
| 时间片轮转 | 同优先级任务时间片切换 |
| 定时器管理 | 软件定时器的基准 |

## 配置

```c
// FreeRTOSConfig.h
#define configTICK_RATE_HZ         (1000)   // 1ms 一次
#define configCPU_CLOCK_HZ          (72000000)  // CPU 主频
```

**注意**：`configTICK_RATE_HZ` 越小，系统开销越小，但延时精度越差。

## xTaskIncrementTick 源码

```c
void xTaskIncrementTick(void) {
    TickType_t xConstTickCount = 0;

    /* 检查是否有任务到期 */
    if (taskTASK_NOT位图 != 0) {
        /* 有任务到期，标记需要切换 */
        xYieldPending = pdTRUE;
    }

    /* 更新时间计数 */
    xConstTickCount = uxTickCount++;
}
```

## 软件定时器 daemon（Timer Service Task）

### 概念

软件定时器是**在指定时间或周期执行回调函数**的机制。

```
[IMAGE_PROMPT: Software timer diagram. Show a timer configured for 1000ms period. Timeline: Timer created at T=0, First expiry at T=1000ms, Second expiry at T=2000ms, Third expiry at T=3000ms. Show timer callback being invoked at each expiry. Label "定时器回调" and "周期 = 1000ms". Use Chinese labels.}
```

### 类型

| 类型 | 说明 |
|------|------|
| **单次定时器** | 到期后停止，只执行一次 |
| **周期定时器** | 到期后自动重置，重复执行 |

### API

```c
// 创建定时器
TimerHandle_t xTimerCreate(
    const char *pcTimerName,         // 名字
    TickType_t xTimerPeriodInTicks,   // 周期（tick 数）
    UBaseType_t uxAutoReload,        // pdTRUE = 周期，pdFALSE = 单次
    void *pvTimerID,                 // 定时器 ID（用于区分多个定时器）
    TimerCallbackFunction_t pxCallbackFunction  // 回调函数
);

// 启动定时器
BaseType_t xTimerStart(TimerHandle_t xTimer, TickType_t xBlockTime);
BaseType_t xTimerStartFromISR(TimerHandle_t xTimer, BaseType_t *pxHigherPriorityTaskWoken);

// 停止定时器
BaseType_t xTimerStop(TimerHandle_t xTimer, TickType_t xBlockTime);

// 重置定时器
BaseType_t xTimerReset(TimerHandle_t xTimer, TickType_t xBlockTime);
```

### 使用示例

```c
// 定时器回调
void timer_callback(TimerHandle_t xTimer) {
    static uint32_t count = 0;
    count++;
    printf("Timer fired: %u\r\n", count);
}

void app_main(void) {
    // 创建周期定时器，1000ms 周期
    TimerHandle_t xTimer = xTimerCreate(
        "1s Timer",
        pdMS_TO_TICKS(1000),  // 1000ms = 1000 ticks
        pdTRUE,               // 周期模式
        NULL,                 // ID
        timer_callback       // 回调函数
    );

    if (xTimer != NULL) {
        xTimerStart(xTimer, 0);  // 启动
    }
}
```

## 软件定时器注意事项

### 1. 回调函数不能阻塞

```c
// ❌ 错误：回调中不能阻塞
void bad_callback(TimerHandle_t xTimer) {
    vTaskDelay(pdMS_TO_TICKS(100));  // ❌ 不能在回调中延时！
}

// ✅ 正确：回调必须快速返回
void good_callback(TimerHandle_t xTimer) {
    uint32_t now = xTaskGetTickCount();  // 只做简单操作
    process_flag = 1;
}
```

### 2. 定时器 daemon 优先级

```c
// FreeRTOSConfig.h
#define configTIMER_TASK_PRIORITY   (2)  // 定时器 daemon 优先级
#define configTIMER_QUEUE_LENGTH   (10)   // 定时器命令队列长度
```

### 3. 定时器 API 实际是队列操作

```c
// xTimerStart 实际上是把命令放入定时器队列
// 由定时器 daemon 从队列取出并执行
BaseType_t xTimerStart(TimerHandle_t xTimer, TickType_t xBlockTime) {
    BaseType_t xReturn;
    xMessage = { tmrCOMMAND_START, ... };
    xReturn = xQueueSend(xTimerHandle->xTimerQueue, &xMessage, xBlockTime);
    return xReturn;
}
```

## 本节小结

- **Tick 中断**是 FreeRTOS 的时间基准，驱动延时和定时器
- **软件定时器**由定时器 daemon 执行，回调函数不能阻塞
- **单次 vs 周期**：创建时 `uxAutoReload` 参数决定
- **定时器 API 实际是队列操作**：命令发给 daemon 执行
