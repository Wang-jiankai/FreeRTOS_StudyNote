## 核心数据结构：TCB（Task Control Block）

每个任务对应一个 `TCB`（任务控制块），定义在 tasks.c：

```c
typedef struct TaskDef {
    volatile StackType_t *pxTopOfStack;  // 栈顶指针（最重要）⚠️

    ListItem_t *xStateListItem;          // 指向任务所在链表节点
    ListItem_t *xEventListItem;          // 事件链表节点
    UBaseType_t uxPriority;              // 当前优先级
    StackType_t *pxStack;                // 栈起始地址
    char pcTaskName[configMAX_TASK_NAME_LEN];  // 任务名

    #if (configUSE_MUTEXES == 1)
        UBaseType_t uxBasePriority;      // 基础优先级（用于优先级继承）
    #endif

    #if (configUSE_TASK_NOTIFICATIONS == 1)
        uint32_t ulNotifiedValue;        // 任务通知值
        uint8_t ucNotifyState;            // 通知状态
    #endif
} tskTaskControlBlock;
```

## xTaskCreate 源码 trace

### Step 1：入参检查

```c:freertos_tasks.c
BaseType_t xTaskCreate(TaskFunction_t pvTaskCode,
                        const char *const pcName,
                        const uint32_t ulStackDepth,
                        void *const pvParameters,
                        UBaseType_t uxPriority,
                        TaskHandle_t *const pxCreatedTask)
{
    Task_t *pxNewTCB;

    /* ---------- 栈大小合法性检查 ---------- */
    if (ulStackDepth > portMAX_STACK_SIZE) {
        return pdFALSE;
    }

    /* ---------- 分配 TCB ---------- */
    pxNewTCB = (Task_t *)pvPortMalloc(sizeof(Task_t));
    if (pxNewTCB == NULL) {
        return errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY;  // 分配失败
    }

    /* 初始化 TCB 成员为 0 */
    memset(pxNewTCB, 0x00, sizeof(Task_t));

    /* ---------- 分配栈 ---------- */
    pxNewTCB->pxStack = (StackType_t *)pvPortMalloc(ulStackDepth * sizeof(StackType_t));
    if (pxNewTCB->pxStack == NULL) {
        vPortFree(pxNewTCB);  // 栈分配失败，释放 TCB
        return errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY;
    }

    /* ---------- 填充 TCB ---------- */
    pxNewTCB->pcTaskName = ...;
    pxNewTCB->uxPriority = uxPriority;
    pxNewTCB->pxTopOfStack = pxPortInitialiseStack(...);  // 初始化栈
    // ...
}
```

### Step 2：栈初始化（关键⚠️）

```c:port.c（Cortex-M 移植层）
StackType_t *pxPortInitialiseStack(StackType_t *pxTopOfStack,
                                    TaskFunction_t pxCode,
                                    void *pvParameters)
{
    /* 模拟中断发生时栈帧的布局 */
    pxTopOfStack--;  // xPSR
    *pxTopOfStack = portINITIAL_XPSR;

    pxTopOfStack--;  // PC（程序计数器）
    *pxTopOfStack = ((StackType_t)pxCode) | portTHUMB_MODE_BIT;

    pxTopOfStack--;  // LR（链接寄存器）
    *pxTopOfStack = (StackType_t)prvTaskExitError;  // 任务退出函数

    /* R0-R12 通用寄存器初始化为 0 */
    for (int i = 12; i >= 0; i--) {
        pxTopOfStack--;
        *pxTopOfStack = 0;
    }

    /* pvParameters 放入 R0 */
    pxTopOfStack--;  /* R0 */
    *pxTopOfStack = (StackType_t)pvParameters;

    return pxTopOfStack;  // 返回新的栈顶
}
```

**为什么这样初始化**：
- 任务第一次被调度器选中时，会触发一次"伪中断"
- CPU 自动从栈中弹出寄存器，恢复任务执行
- 初始化栈帧就是模拟中断发生前的 CPU 状态

## vTaskDelay 源码 trace

### 入口

```c
void vTaskDelay(TickType_t xTicksToDelay)
{
    BaseType_t xShouldYield = pdFALSE;

    /* 延时 0 = yield，让出 CPU 但不阻塞 */
    if (xTicksToDelay > 0) {
        vTaskSuspendAll();  // 挂起调度器，防止切换

        /* 将任务从就绪队列移到延时队列 */
        prvAddCurrentTaskToDelayedList(xTicksToDelay);

        xShouldYield = xTaskResumeAll();  // 恢复调度器
    }

    if (xShouldYield == pdTRUE) {
        taskYIELD();  // 触发 PendSV，强制上下文切换
    }
}
```

### 放入延时队列

```c
static void prvAddCurrentTaskToDelayedList(TickType_t xTicksToWait)
{
    TickType_t xTimeToWake;

    /* 计算唤醒时刻（当前 tick + 延时） */
    xTimeToWake = xTaskGetTickCount() + xTicksToWait;

    /* 构造链表节点 */
    vListInsert(pxDelayedTaskList, &(pxCurrentTCB->xStateListItem));

    /* 标记为阻塞态 */
    listSET_LIST_ITEM_OWNER(&(pxCurrentTCB->xStateListItem), pxCurrentTCB);
}
```

### 唤醒时机

Tick 中断（每 `configTICK_RATE_HZ` ms 一次）中检查延时队列：


## 上下文切换（vTaskSwitchContext）trace

### 触发时机

- `taskYIELD()` → 触发 PendSV 中断
- Tick 中断中判断需要切换

### 核心逻辑

```c
void vTaskSwitchContext(void)
{
    if (listCURRENT_LIST_LENGTH(&pxReadyTasksLists[configMAX_PRIORITIES - 1]) > 0) {
        /* 最高优先级有任务，直接切换 */
        listGET_OWNER_OF_NEXT_ENTRY(pxCurrentTCB, &pxReadyTasksLists[configMAX_PRIORITIES - 1]);
    } else {
        /* 遍历找到最高优先级非空队列 */
        for (uxPriority = configMAX_PRIORITIES - 2; uxPriority >= 0; uxPriority--) {
            if (listCURRENT_LIST_LENGTH(&pxReadyTasksLists[uxPriority]) > 0) {
                listGET_OWNER_OF_NEXT_ENTRY(pxCurrentTCB, &pxReadyTasksLists[uxPriority]);
                break;
            }
        }
    }
}
```

**核心**：从优先级最高的非空就绪队列中取出下一个任务。

## 本节小结

- **TCB** 是任务的核心数据结构，`pxTopOfStack` 是最关键的字段
- **栈初始化** 模拟中断栈帧，确保任务"认为"自己刚被中断恢复
- **就绪队列** 是优先级数组 + 链表，`vTaskDelay` 把任务移入延时链表
- **上下文切换** 从就绪队列找最高优先级任务，恢复其栈帧
