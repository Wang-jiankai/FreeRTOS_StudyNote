## 核心数据结构：TCB（Task Control Block）

### 真实定义（tasks.c）

```c:tasks.c
typedef struct tskTaskControlBlock       /* The old naming convention is used. */
{
    /* MUST BE FIRST - pxTopOfStack 是最关键的字段 */
    volatile StackType_t * pxTopOfStack;

    #if ( portUSING_MPU_WRAPPERS == 1 )
        xMPU_SETTINGS xMPUSettings;  /* MPU 相关，必须是第二个成员 */
    #endif

    ListItem_t xStateListItem;                  /* 任务所在状态链表节点 */
    ListItem_t xEventListItem;                  /* 事件链表节点 */
    UBaseType_t uxPriority;                     /* 当前优先级（0 最低） */
    StackType_t * pxStack;                      /* 栈起始地址 */

    char pcTaskName[ configMAX_TASK_NAME_LEN ]; /* 任务名（静态数组） */

    #if ( configUSE_MUTEXES == 1 )
        UBaseType_t uxBasePriority;             /* 基础优先级（优先级继承用） */
        UBaseType_t uxMutexesHeld;             /* 该任务持有的互斥量数量 */
    #endif

    #if ( configUSE_TASK_NOTIFICATIONS == 1 )
        /* v10.4+ 支持数组，每个任务可有多个通知 */
        volatile uint32_t ulNotifiedValue[ configTASK_NOTIFICATION_ARRAY_ENTRIES ];
        volatile uint8_t ucNotifyState[ configTASK_NOTIFICATION_ARRAY_ENTRIES ];
    #endif

    /* ... 更多条件编译字段 ... */
} tskTCB;

typedef tskTCB TCB_t;  /* 两个名字都常用 */
```

**关键理解**：
- `pxTopOfStack` **必须**是结构体第一个成员（上下文切换汇编代码直接偏移 0 访问）
- 任务通知值是**数组**（v10.4+），每个任务支持多个独立通知
- TCB 本身**不包含栈**，栈是独立分配的，`pxStack` 指向栈底

### 实际调用流程（tasks.c）

```c:tasks.c
BaseType_t xTaskCreate(TaskFunction_t pxTaskCode,
                        const char * const pcName,
                        const configSTACK_DEPTH_TYPE uxStackDepth,
                        void * const pvParameters,
                        UBaseType_t uxPriority,
                        TaskHandle_t * const pxCreatedTask)
{
    TCB_t * pxNewTCB;

    /* 1. 创建任务内部结构（分配栈 + TCB） */
    pxNewTCB = prvCreateTask(pxTaskCode, pcName, uxStackDepth,
                             pvParameters, uxPriority, pxCreatedTask);

    if (pxNewTCB != NULL) {
        /* 2. 加入就绪链表 */
        prvAddNewTaskToReadyList(pxNewTCB);
        xReturn = pdPASS;         /* ✅ 成功返回值是 pdPASS */
    } else {
        xReturn = errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY;  /* -1 */
    }

    return xReturn;
}
```

### 任务内部创建 prvCreateTask（tasks.c）

```c
static TCB_t * prvCreateTask(
    TaskFunction_t pxTaskCode,
    const char * const pcName,
    const configSTACK_DEPTH_TYPE uxStackDepth,
    void * const pvParameters,
    UBaseType_t uxPriority,
    TaskHandle_t * const pxCreatedTask)
{
    TCB_t * pxNewTCB;

    /* 1. 分配 TCB 内存 */
    pxNewTCB = (TCB_t *) pvPortMalloc(sizeof(TCB_t));

    /* 2. 分配栈内存（独立于 TCB） */
    pxNewTCB->pxStack = (StackType_t *) pvPortMalloc(
        (size_t) uxStackDepth * sizeof(StackType_t));

    /* 3. 初始化栈帧（构造模拟中断栈帧） */
    pxNewTCB->pxTopOfStack = pxPortInitialiseStack(
        pxNewTCB->pxStack,
        pxNewTCB->pxStack + uxStackDepth,
        pxTaskCode,
        pvParameters,
        xRunPrivileged,
        pxMPUSettings);

    /* 4. 初始化任务名、优先级、链表节点 */
    /* ... */
}
```

## pxPortInitialiseStack（port.c，移植层）

> 注意：此函数是**移植层实现**，不同架构代码不同。以下是 Cortex-M 的实现。

```c:port.c（ARM_CM4F/ARM_CM3/ARM_CM7）
StackType_t * pxPortInitialiseStack(StackType_t * pxTopOfStack,
                                    StackType_t * pxStackEnd,
                                    TaskFunction_t pxTaskCode,
                                    void * pvParameters,
                                    BaseType_t xRunPrivileged,
                                    xMPU_SETTINGS * pxMPUSettings)
{
    pxTopOfStack--;

    *pxTopOfStack = portINITIAL_XPSR;            /* xPSR = 0x01000000（Thumb bit 必须） */
    pxTopOfStack--;
    *pxTopOfStack = (StackType_t) pxTaskCode;   /* PC = 任务入口地址 */

    /* 注意：这里不存退出函数！栈帧中 LR 是异常返回值，不是退出函数 */

    /* 跳过 R12, R3, R2, R1（共 4 个寄存器，硬件自动压） */
    pxTopOfStack -= 5;

    *pxTopOfStack = (StackType_t) pvParameters;   /* R0 = 任务参数 */

    pxTopOfStack--;
    *pxTopOfStack = portINITIAL_EXC_RETURN;     /* LR = 0xFFFFFFFD（返回后使用 PSP） */

    pxTopOfStack -= 8;  /* R4-R11 由 PendSV_Handler 软件保存，不是这里 */

    return pxTopOfStack;
}
```

**为什么这里不保存 R4-R11**：
- 中断发生时，CPU **硬件自动**压入 R0-R3, R12, LR, PC, xPSR（8 个寄存器）
- R4-R11 由 PendSV_Handler 中的**软件代码**保存/恢复
- `pxPortInitialiseStack` 只需要构造"中断刚要发生时"的栈状态

## vTaskDelay 源码 trace

```c:tasks.c
void vTaskDelay(TickType_t xTicksToDelay)
{
    if (xTicksToDelay > 0) {
        vTaskSuspendAll();           /* 挂起调度器（禁止任务切换，不关中断） */

        /* 计算唤醒 tick 值，加入延时链表 */
        prvAddCurrentTaskToDelayedList(xTicksToDelay);

        if (xTaskResumeAll() == pdFALSE) {
            portYIELD();             /* 需要切换，发起 PendSV */
        }
    }
}
```

## 本节小结

- **TCB**（`tskTCB`/`TCB_t`）：`pxTopOfStack` 必须第一个成员，`uxBasePriority` 用于优先级继承
- **xTaskCreate** 返回 `pdPASS`（成功）或 `errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY`（-1）
- **栈初始化**（`pxPortInitialiseStack`）：构造模拟中断栈帧，`LR=0xFFFFFFFD` 表示返回后使用 PSP
- **就绪队列**：`pxReadyTasksLists[]` 是按优先级索引的链表数组
- R4-R11 由 PendSV_Handler 软件保存，不在 `pxPortInitialiseStack` 中
