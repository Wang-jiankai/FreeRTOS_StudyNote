# 1.1 概念：任务状态机、优先级、栈

## 任务（Task）是什么

任务是 FreeRTOS 调度的基本单位。每个任务本质上是一个无限循环的函数：

```c
void my_task(void *param) {
    while (1) {
        do_something();
        vTaskDelay(pdMS_TO_TICKS(10));  // 必须阻塞，否则高优先级任务无法运行
    }
}
```

---

## 任务状态机

FreeRTOS 中的任务有 **4 种状态**：

| 状态 | 含义 | CPU 占用 |
|------|------|---------|
| **Running** | 正在执行 | ✅ 占用 |
| **Ready** | 就绪（等待 CPU） | ❌ 不占用 |
| **Blocked** | 阻塞（等待事件） | ❌ 不占用 |
| **Suspended** | 挂起（显式暂停） | ❌ 不占用 |

<img src="./Gemini_Generated_Image_ojw9z0ojw9z0ojw9.png" width="800" alt="任务状态机">

### 状态详解

#### Running（运行态）
- CPU 正在执行这个任务
- 同一时刻只有一个任务处于 Running 态（单核情况下）

#### Ready（就绪态）
- 具备运行条件，等待 CPU 空闲
- 调度器选中的下一个运行的总是 Ready 队列中最高优先级的任务

#### Blocked（阻塞态）
- 任务在等待某个事件：
  - `vTaskDelay()` 等待延时结束
  - `xQueueReceive()` 等待队列数据
  - `xSemaphoreTake()` 等待信号量
- 不消耗 CPU

#### Suspended（挂起态）
- 任务被显式调用 `vTaskSuspend()` 暂停
- 只有调用 `vTaskResume()` 才能恢复
- 不参与调度

## 优先级（Priority）

### 概念

- 优先级范围：`0 ~ (configMAX_PRIORITIES - 1)`
- **数字越大，优先级越高**
- `configMAX_PRIORITIES` 默认 5，可在 `FreeRTOSConfig.h` 中修改

<img src="./Gemini_Generated_Image_oqiso5oqiso5oqis.png" width="800" alt="优先级">

### 优先级反转问题

低优先级任务持有高优先级需要的资源，导致高优先级任务反而无法运行。

<img src="./Gemini_Generated_Image_jxi3wdjxi3wdjxi3.png" width="800" alt="优先级反转">

**解决**：FreeRTOS 提供**优先级继承**机制（见第 5 章同步原语）。

## 任务栈（Stack）

### 概念

每个任务有独立的栈空间，用于保存：
- 函数局部变量
- 函数调用返回地址
- 中断时保存 CPU 寄存器

### 栈大小单位

**注意**：FreeRTOS 栈大小以 **字（word）** 为单位，不是字节。

- ARM Cortex-M 是 32 位 = 4 字节/字
- `configMINIMAL_STACK_SIZE = 128` 表示 128 × 4 = **512 字节**

### 如何计算栈大小

```c
// 创建任务时指定栈大小
xTaskCreate(task_func, "TaskName", 256, NULL, 1, NULL);
//                                          ^^^
//                                          256 字 = 1024 字节
```

### 栈溢出检测

FreeRTOS 提供两种检测方式（`FreeRTOSConfig.h` 配置）：

| 模式 | 配置 | 检测方式 |
|------|------|---------|
| 模式 1 | `configCHECK_FOR_STACK_OVERFLOW = 1` | 任务切换时检测栈指针是否在有效范围 |
| 模式 2 | `configCHECK_FOR_STACK_OVERFLOW = 2` | 任务切换时检测栈 magic value（`0xA5A5A5A5`）是否被破坏 |

```c
// 栈溢出钩子函数（需用户实现）
void vApplicationStackOverflowHook(TaskHandle_t xTask, char *pcTaskName) {
    printf("Stack overflow in task: %s\r\n", pcTaskName);
}
```

---

## 本节小结

- 任务有 4 种状态：Running / Ready / Blocked / Suspended
- 优先级数字越大越高，抢占式调度保证高优先级任务优先运行
- 每个任务有独立栈，栈大小以字为单位
- 栈溢出是常见 bug，启用检测钩子函数可帮助排查
