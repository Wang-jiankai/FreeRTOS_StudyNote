# 1.2 API 速查：任务创建 / 删除 / 挂起

## 任务创建

### xTaskCreate（动态创建）✅ 最常用

```c
BaseType_t xTaskCreate(
    TaskFunction_t pvTaskCode,    // 任务函数指针（必须是无返回值参数为 void* 的函数）
    const char *pcName,          // 任务名称（仅用于调试，最大 16 字符含结尾 \0）
    configSTACK_DEPTH_TYPE usStackDepth, // 栈大小（单位是 stack Type_t，通常是 uint16_t）
    void *pvParameters,          // 传给任务函数的参数（可为 NULL）
    UBaseType_t uxPriority,       // 任务优先级（0 ~ configMAX_PRIORITIES-1）
    TaskHandle_t *pxCreatedTask   // 输出任务句柄（可用于删除/挂起任务，可为 NULL）
);
```

**返回值**：`pdPASS`（成功）/ `errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY`（内存不足，值为 -1）

**示例**：

```c
void sensor_task(void *param) {
    uint32_t interval = *(uint32_t *)param;  // 使用传入参数
    while (1) {
        read_sensor();
        vTaskDelay(pdMS_TO_TICKS(interval));
    }
}

int main(void) {
    uint32_t interval_100ms = 100;
    xTaskCreate(sensor_task, "Sensor", 256, &interval_100ms, 2, NULL);
    xTaskCreate(display_task, "Display", 256, NULL, 1, NULL);
    vTaskStartScheduler();
}
```

### xTaskCreateStatic（静态创建）⚠️ 很少用

```c
TaskHandle_t xTaskCreateStatic(
    TaskFunction_t pvTaskCode,
    const char *pcName,
    uint32_t ulStackDepth,
    void *pvParameters,
    UBaseType_t uxPriority,
    StackType_t *pxStackBuffer,    // 需用户预分配栈内存
    StaticTask_t *pxTaskBuffer      // 需用户预分配 TCB 内存
);
```

**何时用**：不想用动态内存分配时。嵌入式场景很少用。

## 任务删除

### vTaskDelete

```c
void vTaskDelete(TaskHandle_t xTaskToDelete);
```

**注意**：
- 删除自己：`vTaskDelete(NULL)`（推荐）或 `vTaskDelete(xTaskGetCurrentTaskHandle())`
- 删除任务会释放任务的栈和 TCB（动态分配的内存）
- **不会自动释放任务持有的信号量/锁**（容易漏，是常见 bug）

```c
// 错误示例：任务持有一个信号量，被删除时没释放
void bad_task(void *param) {
    xSemaphoreTake(mutex, portMAX_DELAY);  // 获取互斥量
    do_work();
    vTaskDelete(NULL);  // 互斥量没释放，其他任务永远拿不到
}

// 正确做法：清理函数中释放资源
void good_task(void *param) {
    xSemaphoreTake(mutex, portMAX_DELAY);
    do_work();
    xSemaphoreGive(mutex);  // 先释放
    vTaskDelete(NULL);
}
```

## 任务延时（阻塞自己）

### vTaskDelay（相对延时）

```c
void vTaskDelay(TickType_t xTicksToDelay);
// 宏：
TickType_t pdMS_TO_TICKS(uint32_t xMilliseconds);  // 毫秒 → tick 数
```

**特点**：
- 延时是**相对于调用时刻**的
- 调用后任务进入 Blocked 态，释放 CPU
- 不精确（唤醒时间 = 调用时刻 + 延时，受调度器影响）

```c
// 例：每 100ms 执行一次
void task(void *param) {
    while (1) {
        do_work();
        vTaskDelay(pdMS_TO_TICKS(100));  // 实际间隔 ≈ 100ms + 调度开销
    }
}
```

### vTaskDelayUntil（绝对延时）✅ 更精确

```c
void vTaskDelayUntil(TickType_t *pxPreviousWakeTime, TickType_t xTimeIncrement);
```

**特点**：
- 延时是**相对于上一次唤醒时刻**的
- 保证固定周期，抖动小
- 常用于周期性任务

```c
// 例：严格每 100ms 执行一次
void precise_task(void *param) {
    TickType_t previous_wake = xTaskGetTickCount();
    while (1) {
        do_work();
        vTaskDelayUntil(&previous_wake, pdMS_TO_TICKS(100));  // 绝对延时
    }
}
```

| 特性 | vTaskDelay | vTaskDelayUntil |
|------|------------|----------------|
| 周期性任务 | ❌ 抖动累积 | ✅ 精确周期 |
| 第一次执行 | 立即 | 立即 |
| 适用场景 | 非周期性任务 | 周期性任务 |

## 任务挂起 / 恢复

### vTaskSuspend（挂起）

```c
void vTaskSuspend(TaskHandle_t xTaskToSuspend);
```

- 被挂起的任务**不参与调度**，直到被 resume
- 挂起时不释放任何资源（信号量、锁等）
- 可挂起自己：`vTaskSuspend(NULL)`

### vTaskResume（恢复）

```c
void vTaskResume(TaskHandle_t xTaskToResume);
```

- 将挂起的任务重新放入就绪队列

### vTaskResumeFromISR（中断中恢复）⚠️ 少用

```c
BaseType_t xTaskResumeFromISR(TaskHandle_t xTaskToResume);
// 返回值：pdTRUE 表示恢复后可能需要触发一次上下文切换
```

## 常用 API 速查表

| 函数 | 说明 | 返回值 |
|------|------|--------|
| `xTaskCreate()` | 动态创建任务 | pdPASS/err |
| `vTaskDelete()` | 删除任务 | void |
| `vTaskDelay()` | 相对延时 | void |
| `vTaskDelayUntil()` | 绝对延时（周期性） | void |
| `vTaskSuspend()` | 挂起任务 | void |
| `vTaskResume()` | 恢复任务 | void |
| `vTaskPrioritySet()` | 设置优先级 | void |
| `uxTaskPriorityGet()` | 获取优先级 | UBaseType_t |
| `xTaskGetCurrentTaskHandle()` | 获取当前任务句柄 | TaskHandle_t |
| `xTaskGetTickCount()` | 获取当前 tick 数 | TickType_t |

---

## 本节小结

- `xTaskCreate()` 是最常用的任务创建方式
- `vTaskDelay()` vs `vTaskDelayUntil()`：周期性任务用后者
- `vTaskDelete()` 要注意先释放资源（信号量、锁）
- 挂起/恢复不常用，但在某些暂停场景有用
