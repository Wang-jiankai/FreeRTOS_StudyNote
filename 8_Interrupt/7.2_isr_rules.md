# 7.2 FromISR 的意义：中断安全函数

## 为什么需要 FromISR 版本

普通 FreeRTOS API 可能会**阻塞**（等待资源），而中断上下文**不能阻塞**。

```
[IMAGE_PROMPT: Comparison diagram showing two paths. Left: Task calling xQueueSend - can block if queue full. Right: ISR calling xQueueSendFromISR - never blocks, returns immediately. Show the different outcomes. Use Chinese labels "任务上下文（可阻塞）" and "中断上下文（不可阻塞）".}
```

### 对比示例

| 任务版本 | FromISR 版本 |
|---------|-------------|
| `xQueueSend()` | `xQueueSendFromISR()` |
| `xQueueReceive()` | `xQueueReceiveFromISR()` |
| `xSemaphoreTake()` | `xSemaphoreTakeFromISR()` |
| `xEventGroupSetBits()` | `xEventGroupSetBitsFromISR()` |

## FromISR 的限制

### 不能阻塞

```c
// ❌ 错误：FromISR 不能用 portMAX_DELAY
void UART_IRQHandler(void) {
    xQueueReceiveFromISR(xQueue, &data, portMAX_DELAY);  // ❌ 危险！
}

// ✅ 正确：FromISR 立即返回
BaseType_t xHigherPriorityTaskWoken = pdFALSE;
xQueueReceiveFromISR(xQueue, &data, &xHigherPriorityTaskWoken);
```

### 返回值不同

```c
// 任务版：pdTRUE 成功，errQUEUE_FULL 失败
BaseType_t result = xQueueSend(xQueue, &data, portMAX_DELAY);

// FromISR 版：pdTRUE 有更高优先级任务就绪，pdFALSE 没有
BaseType_t result = xQueueSendFromISR(xQueue, &data, &xHigherPriorityTaskWoken);
```

---

## xHigherPriorityTaskWoken 机制

### 概念

当 `FromISR` 检测到有更高优先级任务被唤醒时，通过这个参数通知调用者。

```
[IMAGE_PROMPT: Task wakeup flow in FromISR. Show interrupt fires, FromISR is called, which wakes a high-priority task, sets xHigherPriorityTaskWoken=pdTRUE. Then after ISR exits, portYIELD_FROM_ISR triggers scheduler. Label each step. Use Chinese labels.}
```

### 必须的检查

```c
void UART_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    // 从队列接收数据
    uint8_t ch;
    xQueueReceiveFromISR(xUartQueue, &ch, &xHigherPriorityTaskWoken);

    // ✅ 关键：判断是否需要触发调度
    if (xHigherPriorityTaskWoken == pdTRUE) {
        portYIELD_FROM_ISR();
    }
}
```

### 为什么需要手动 yield？

- 中断中不能直接调用 `vTaskSwitchContext()`
- `portYIELD_FROM_ISR()` 标记需要切换，等中断返回后由调度器执行

## 实战：完整中断处理

### 场景：UART 接收 + 任务处理

```c
// 串口接收队列（任务等待数据）
QueueHandle_t xUartRxQueue;

// 中断：接收字节，放入队列
void USART1_IRQHandler(void) {
    uint8_t ch;

    // 接收数据（寄存器读取）
    ch = USART1->DR;

    // 发送到队列
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xQueueSendFromISR(xUartRxQueue, &ch, &xHigherPriorityTaskWoken);

    // 必要时触发调度
    if (xHigherPriorityTaskWoken == pdTRUE) {
        portYIELD_FROM_ISR();
    }
}

// 任务：处理串口数据
void uart_process_task(void *param) {
    uint8_t ch;
    while (1) {
        if (xQueueReceive(xUartRxQueue, &ch, portMAX_DELAY) == pdTRUE) {
            // 处理接收到的数据
            process_byte(ch);
        }
    }
}
```

---

## 本节小结

- **FromISR** 是中断安全的 API，不能阻塞
- `pxHigherPriorityTaskWoken` 参数用于通知是否有高优先级任务就绪
- 每次 FromISR 调用后必须检查 `pxHigherPriorityTaskWoken` 并调用 `portYIELD_FROM_ISR()`
- FromISR 版本不能使用 `portMAX_DELAY` 等阻塞参数
