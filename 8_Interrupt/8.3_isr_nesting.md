## 优先级组

Cortex-M 使用优先级寄存器：
- 8 位优先级寄存器（实际位数看芯片）
- 数值越小优先级越高
- 未使用的低位可以嵌套

## 中断嵌套配置

### 1. 设置优先级

```c
// 数字越小优先级越高
NVIC_SetPriority(UART_IRQn, 5);    // 较低优先级
NVIC_SetPriority(TIM_IRQn, 3);     // 较高优先级
```

### 2. 启用嵌套

```c
// Cortex-M 默认允许嵌套
// 只需确保高优先级中断能打断低优先级

// 设置基础优先级（分组）
NVIC_SetPriorityGrouping(NVIC_PRIORITYGROUP_4_SUB_4);
```

## FreeRTOS 与中断嵌套

### configMAX_SYSCALL_INTERRUPT_PRIORITY

这是 FreeRTOS 最重要的中断配置：

```c
// FreeRTOSConfig.h
#define configMAX_SYSCALL_INTERRUPT_PRIORITY    5
```

**含义**：优先级数值 > 5 的中断，可以调用 FreeRTOS API。

```
[IMAGE_PROMPT: FreeRTOS interrupt priority zones. Vertical scale from 0 (highest) to 15 (lowest).
Zone 1 (0-4): "受保护区域" - can NOT call FreeRTOS API
Zone 2 (5-15): "安全区域" - can call FromISR API
Mark configMAX_SYSCALL_INTERRUPT_PRIORITY=5 threshold with red line.
Label each zone with examples. Use Chinese labels.}
```

### 为什么这样设计？

```c
// 临界段关闭中断的方式
portDISABLE_INTERRUPTS();  // 只关闭优先级 <= 5 的中断

// 优先级 0-4 的中断不受影响
// → 这些中断仍可执行，保证系统响应
```

## 中断嵌套实战

### 例 1：UART + DMA 双优先级

```c
// 优先级 6：UART 中断（接收字节）
void UART_IRQHandler(void) {
    uint8_t data = UART->DR;
    xQueueSendFromISR(xUartQueue, &data, &xHigherPriorityTaskWoken);

    if (xHigherPriorityTaskWoken) {
        portYIELD_FROM_ISR();
    }
}

// 优先级 4：DMA 中断（传输完成）— 不能调用 FromISR！
void DMA_IRQHandler(void) {
    // 只做最简单的事
    DMA->IFCR |= DMA_IFCR_CTCIF1;
}
```

### 例 2：定时器更新 + 按钮中断

```c
// 优先级 5：定时器更新（调用 FreeRTOS API）
void TIM2_IRQHandler(void) {
    if (TIM2->SR & TIM_SR_UIF) {
        TIM2->SR &= ~TIM_SR_UIF;
        xSemaphoreGiveFromISR(xTimerSem, NULL);  // ✅ 安全
    }
}

// 优先级 3：按钮中断（紧急，不调 API）
void EXTI0_IRQHandler(void) {
    // 紧急处理，不调用 FreeRTOS API
    critical_flag = 1;
    // 清除中断
}
```

## 嵌套常见错误

### 错误 1：高优先级中断调 API

```c
// ❌ 优先级 2 < 5，危险！
NVIC_SetPriority(IRQ_X, 2);
void IRQ_X_Handler(void) {
    xQueueSendFromISR(xQ, &data, NULL);  // ❌ 可能出问题
}

// ✅ 正确：优先级要 > 5
NVIC_SetPriority(IRQ_X, 6);
```

### 错误 2：忘记在 FromISR 后 yield

```c
// ❌ 任务可能不会立即切换
void UART_IRQHandler(void) {
    xQueueSendFromISR(xQ, &data, NULL);  // 没检查返回值
}

// ✅ 正确：必须检查并 yield
void UART_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xQueueSendFromISR(xQ, &data, &xHigherPriorityTaskWoken);

    if (xHigherPriorityTaskWoken) {
        portYIELD_FROM_ISR();
    }
}
```

### 错误 3：中断优先级设错导致无法响应

```c
// ❌ 优先级 15 = 最低，无法打断任何东西
NVIC_SetPriority(重要中断, 15);

// ✅ 正确：确保优先级够高
NVIC_SetPriority(重要中断, 2);
```

## 中断与临界段的关系

| 操作 | 影响范围 |
|------|---------|
| `portDISABLE_INTERRUPTS()` | 优先级 ≤ `configMAX_SYSCALL_INTERRUPT_PRIORITY` |
| `taskENTER_CRITICAL()` | 同上 + 嵌套计数 |
| 高优先级中断（0-4） | 不受任何影响 |

## 本节小结

- Cortex-M 优先级数值越小越高
- 中断嵌套：高优先级可打断低优先级
- `configMAX_SYSCALL_INTERRUPT_PRIORITY` 定义了可以调用 FreeRTOS API 的最低优先级
- 优先级 0-4 的中断不受 FreeRTOS 临界段影响
- 每个 `FromISR` 调用后必须检查 `xHigherPriorityTaskWoken` 并可能 yield
