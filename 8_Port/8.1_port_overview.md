# 8.1 移植层结构：portmacro.h

## 为什么需要移植层

FreeRTOS 是一款跨平台的 RTOS，但不同芯片的架构差异很大：
- 上下文切换的实现不同
- 中断控制器不同
- 汇编语法不同

移植层负责**适配具体芯片**，让同一套内核代码能运行在不同硬件上。

## 移植层文件

| 文件 | 作用 |
|------|------|
| `portmacro.h` | 宏定义（数据类型、临界段、堆栈对齐等） |
| `port.c` | 核心函数（任务启动、上下文切换、Tick 配置等） |
| `portmacro.h` | 汇编实现（PendSV_Handler 等） |

## 核心移植配置（portmacro.h）

### 数据类型

```c
// FreeRTOS 使用固定宽度的数据类型
#define portSTACK_TYPE          uint32_t
#define portBASE_TYPE           long
#define portUBaseType_t        uint32_t
#define portTickType           uint32_t
```

### 堆栈对齐

```c
// ARM Cortex-M 要求 8 字节对齐
#define portBYTE_ALIGNMENT      8
#define portBYTE_ALIGNMENT_MASK (0x0007)
```

### 临界段

```c
#define portDISABLE_INTERRUPTS()    __asm volatile ("cpsid i")
#define portENABLE_INTERRUPTS()     __asm volatile ("cpsie i")

#define portENTER_CRITICAL()        vPortEnterCritical()
#define portEXIT_CRITICAL()         vPortExitCritical()
```

### YIELD

```c
// 触发 PendSV 中断（上下文切换）
#define portYIELD() \
    do { \
        portNVIC_INT_CTRL_REG = portNVIC_PENDSVSET_BIT; \
        __asm volatile ("dsb" ::: "memory"); \
        __asm volatile ("isb"); \
    } while (0)
```

## 核心函数

### vPortStartFirstTask

第一个任务启动时调用：

```c
void vPortStartFirstTask(void) {
    __asm volatile (
        " LDR R0, =0xE000ED04  \n"  // Vector table offset
        " LDR R0, [R0]             \n"  // Get MSP
        " MSR msp, R0               \n"  // Set MSP
        " CPSIE i                   \n"  // Enable interrupts
        " SVC #0                    \n"  // Trigger SVC (starts scheduler)
    );
}
```

### PendSV_Handler

上下文切换的中断处理：

```c
void PendSV_Handler(void) {
    __asm volatile (
        /* 保存上下文 */
        " MRS    r0, PSP             \n"
        " SUBS   r0, r0, #32         \n"
        " STM    r0!, {r4-r11}       \n"

        /* 调用调度器 */
        " BL     vTaskSwitchContext   \n"

        /* 恢复新任务上下文 */
        " LDR    r0, =pxCurrentTCB  \n"
        " LDR    r0, [r0]             \n"
        " LDR    r1, [r0]             \n"
        " LDM    r1!, {r4-r11}       \n"
        " ADDS   r1, r1, #32         \n"
        " MSR    PSP, r1             \n"
        " BX     lr                   \n"
    );
}
```

## 获取 FreeRTOS 移植层

### 方式 1：官方包自带

```
FreeRTOS/Source/portable/
├── ARM_CM3/       // Cortex-M3
├── ARM_CM4F/       // Cortex-M4F (带浮点)
├── ARM_CM7/        // Cortex-M7
├── GCC/ARM_CM3/    // GCC 编译器
├── IAR/ARM_CM3/   // IAR 编译器
└── ...
```

### 方式 2：芯片厂商提供

ST（STM32）、NXP、TIVA 等厂商会在 SDK 中提供适配好的 FreeRTOS 移植层。

## 本节小结

- **移植层**是 FreeRTOS 和具体芯片之间的适配层
- 核心文件是 `portmacro.h`（宏）和 `port.c`（函数）
- `portmacro.h` 定义数据类型、临界段、堆栈对齐等
- `port.c` 实现任务启动、上下文切换（PendSV_Handler）
- 获取移植层：FreeRTOS 官方包或芯片厂商 SDK
