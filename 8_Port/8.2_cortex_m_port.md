# 8.2 ARM Cortex-M 移植要点

## Cortex-M 移植核心

| 要素 | 说明 |
|------|------|
| 栈生长方向 | 向低地址生长（满递减） |
| 数据类型 | 32 位 |
| 上下文切换 | PendSV 中断 |
| 中断优先级 | 数值越小优先级越高 |
| 栈对齐 | 8 字节对齐 |

## 栈初始化

```c
StackType_t *pxPortInitialiseStack(
    StackType_t *pxTopOfStack,
    TaskFunction_t pxCode,
    void *pvParameters)
{
    /* 模拟中断栈帧 */
    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)portINITIAL_XPSR;  // xPSR = 1

    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)pxCode | portTHUMB_MODE_BIT;  // PC = 任务入口

    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)prvTaskExitError;  // LR = 退出函数

    /* R0-R12 初始化为 0 */
    for (int i = 12; i >= 0; i--) {
        pxTopOfStack--;
        *pxTopOfStack = 0;
    }

    /* R0 = 任务参数 */
    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)pvParameters;

    return pxTopOfStack;
}
```

## 上下文切换机制

### 触发方式

```c
// 方式 1：SVC 触发（常用）
#define portYIELD() __asm volatile ("svc 0")

// 方式 2：直接触发 PendSV（等效）
#define portYIELD() __asm volatile ("movw r0, 0xEDFF; movt r0, 0xE000; ldr r0, [r0]; str r0, [r0]")
```

### PendSV 优先级

```c
// 在 xPortStartScheduler 中设置
NVIC_SetPriority(PendSV_IRQn, 0xFF);  // 最低优先级
```

**为什么 PendSV 设最低优先级？** 确保所有更重要的中断处理完才进行任务切换。

## Cortex-M3/M4/M7 差异

| 特性 | M3 | M4 | M7 |
|------|----|----|-----|
| 架构 | v7-M | v7E-M | v7E-M |
| FPU | ❌ | 可选 | 可选 |
| DSP | ❌ | 可选 | 可选 |
| 移植层 | ARM_CM3 | ARM_CM4F | ARM_CM7 |

### FPU 支持

如果芯片有 FPU，需要使能：

```c
// 在 FreeRTOSConfig.h
#define configFPU_HARDWARE  1  // 使能 FPU

// 在启动代码或 port.c 中
void vPortEnableVFP(void) {
    __asm volatile (
        " LDR.W R0, =0xE000ED88   \n"
        " LDR R1, [R0]            \n"
        " ORR R1, R1, #(1 << 20)  \n"
        " STR R1, [R0]            \n"
        " ISB                      \n"
    );
}
```

## 常见移植问题

### 1. 堆栈对齐错误

```c
// ❌ 错误：堆栈地址不是 8 的倍数
void bad_task(void *p) {
    uint64_t big_var;  // 可能在栈上，但地址可能不对齐
}

// ✅ 正确：手动对齐
void good_task(void *p) {
    __attribute__((aligned(8))) uint64_t big_var;
}
```

### 2. 遗忘 FPU 保存

如果用了 FPU 但没在任务栈帧中保存，会导致数据丢失。

### 3. 中断优先级设错

PendSV 必须设为**最低优先级**（数值最大）。

---

## 本节小结

- Cortex-M 移植核心：PendSV 切换、8字节对齐、栈帧布局
- `pxPortInitialiseStack` 构造初始栈帧，模拟中断刚发生
- PendSV 必须设为最低优先级，确保其他中断先处理
- M4/M7 有可选 FPU，需要额外配置
