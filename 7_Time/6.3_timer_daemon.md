## 定时器命令

| 命令 | 说明 |
|------|------|
| `tmrCOMMAND_START` | 启动定时器 |
| `tmrCOMMAND_STOP` | 停止定时器 |
| `tmrCOMMAND_RESET` | 重置定时器 |
| `tmrCOMMAND_CHANGE_PERIOD` | 更改周期 |
| `tmrCOMMAND_DELETE` | 删除定时器 |

## 源码解析

### 定时器结构体

```c:timers.c
typedef struct tmrTimerControl {
    const char *pcTimerName;           // 名字
    ListItem_t xTimerListItem;         // 链表节点（用于排序）
    TickType_t xTimerPeriodInTicks;   // 周期
    UBaseType_t uxAutoReload;         // 是否自动重载
    void *pvTimerID;                  // 用户 ID
    TimerCallbackFunction_t pxCallbackFunction;  // 回调函数
    /* ... 状态字段 ... */
} xTIMER;
```

### daemon 任务主循环

```c:timers.c
static void prvTimerTask(void *pvParameters) {
    for (;;) {
        /* 等待定时器命令（带阻塞） */
        xTimerReceive = xQueueReceive(
            xTimerQueue,
            &xMessage,
            portMAX_DELAY  // 无限等待
        );

        /* 处理命令 */
        switch (xMessage.xMessageID) {
            case tmrCOMMAND_START:
                prvStartTimer(xMessage.pxTimer);
                break;
            case tmrCOMMAND_STOP:
                prvStopTimer(xMessage.pxTimer);
                break;
            /* ... */
        }
    }
}
```

### 定时器到期检查

```c
static void prvProcessTimerList(void) {
    /* 按到期时间排序的链表 */
    const ListItem_t *pxItem = listGET_HEAD_LIST(&xActiveTimerList);

    while (pxItem != listGET_END_MARKER(&xActiveTimerList)) {
        xTimeNow = xTaskGetTickCount();

        /* 检查是否到期 */
        if (xTimeNow >= listGET_LIST_ITEM_VALUE(pxItem)) {
            xTIMER *pxTimer = listGET_OWNER_OF_HEAD_ENTRY(pxItem);

            /* 调用回调（不传递参数） */
            pxTimer->pxCallbackFunction(pxTimer);

            /* 如果是周期定时器，重置到期时间 */
            if (pxTimer->uxAutoReload == pdTRUE) {
                listSET_LIST_ITEM_VALUE(pxItem,
                    xTimeNow + pxTimer->xTimerPeriodInTicks);
            }
        }
        pxItem = listGET_NEXT(pxItem);
    }
}
```

## 配置参数

```c
// FreeRTOSConfig.h
#define configTIMER_TASK_PRIORITY    (2)      // daemon 优先级
#define configTIMER_QUEUE_LENGTH     (10)     // 命令队列长度
#define configTIMER_TASK_STACK_DEPTH (configMINIMAL_STACK_SIZE)  // 栈大小
```

### 优先级选择

| 场景 | 建议优先级 | 原因 |
|------|----------|------|
| 定时器回调简单快速 | 2（低） | 减少 CPU 占用 |
| 定时器触发重要动作 | 更高 | 确保及时执行 |
| 与其他任务交互 | 参考通信任务 | 避免优先级反转 |

## 与硬件定时器的区别

| 特性 | 软件定时器 | 硬件定时器 |
|------|-----------|-----------|
| 精度 | Tick 级（通常 1ms） | us 级（微秒级，受 MCU 主频限制） |
| 数量 | 无数个（受内存限制） | 有限（2-4 个） |
| CPU 占用 | 仅到期时 | 持续运行 |
| 回调上下文 | daemon 任务 | 中断 |

---

## 本节小结

- 定时器 daemon 是一个任务，通过命令队列接收指令
- 定时器按到期时间排序存储在链表中
- `xTimerStart/Stop/Reset` 实际是向队列发命令
- 软件定时器精度受 Tick 配置影响，不适合高精度场景
