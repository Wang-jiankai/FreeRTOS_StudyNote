## xQueueSend 源码 trace

### Step 1：入口

```c:queue.c
BaseType_t xQueueGenericSend(QueueHandle_t xQueue,
                              const void *pvItemToQueue,
                              TickType_t xTicksToWait,
                              const BaseType_t xCopyPosition)  // xCopyPosition = queueSEND_TO_BACK or queueSEND_TO_FRONT
{
    BaseType_t xEntryTimeSet = pdFALSE;
    TimeOut_t xTimeOut;

    /* 进入临界段（禁止中断） */
    taskENTER_CRITICAL();

    for (;;) {
        /* 检查队列是否有空间 */
        if (uxQueueMessagesWaiting(xQueue) < xQueue->uxLength) {
            /* 有空间，可以写入 */
            break;
        }

        /* 队列满，等待 */
        if (xEntryTimeSet == pdFALSE) {
            vTaskSetTimeOutState(&xTimeOut);  // 记录超时起点
            xEntryTimeSet = pdTRUE;
        }

        /* 当前任务从就绪队列移到等待发送队列 */
        vTaskPlaceOnEventList(&xQueue->xTasksWaitingToSend, xTicksToWait);

        /* 释放 CPU，触发调度 */
        taskEXIT_CRITICAL();
        taskYIELD();
        taskENTER_CRITICAL();

        /* 检查超时 */
        if (xTaskCheckTimeOut(&xTimeOut, &xTicksToWait) == pdTRUE) {
            /* 超时，任务从等待列表移除 */
            taskEXIT_CRITICAL();
            return errQUEUE_FULL;
        }
    }
}
```

### Step 2：拷贝数据到队列

```c:queue.c
    /* 拷贝数据到队列存储区 */
    prvCopyDataToQueue(xQueue, pvItemToQueue, xCopyPosition);
```

```c:queue.c
static void prvCopyDataToQueue(Queue_t * const pxQueue,
                               const void *pvItemToQueue,
                               const BaseType_t xCopyPosition)
{
    if (pxQueue->uxItemSize == (UBaseType_t)0) {
        /* 长度为 0？空队列？直接返回 */
        return;
    }

    if (xCopyPosition == queueSEND_TO_FRONT) {
        /* 发送到头部：移动 pcReadFrom 指针 */
        pxQueue->pcReadFrom = pxQueue->pcHead + (pxQueue->uxLength - 1) * pxQueue->uxItemSize;
    }

    /* 拷贝数据到 pcWriteTo 指向的位置 */
    memcpy(pxQueue->pcWriteTo, pvItemToQueue, pxQueue->uxItemSize);

    /* 更新写入位置（循环） */
    if (pxQueue->uxMessagesWaiting == (UBaseType_t)0) {
        /* 队列之前是空的，唤醒一个等待接收的任务 */
        if (listCURRENT_LIST_LENGTH(&pxQueue->xTasksWaitingToReceive) > 0) {
            /* 从等待接收列表取出第一个任务 */
            if (xTaskRemoveFromEventList(&pxQueue->xTasksWaitingToReceive) == pdTRUE) {
                /* 需要触发调度 */
                vTaskMissedYield();
            }
        }
    }

    /* 增加消息计数 */
    (pxQueue->uxMessagesWaiting)++;

    /* 移动 pcWriteTo 指针（循环队列） */
    if (pxQueue->pcWriteTo >= (pxQueue->pcHead + pxQueue->uxLength * pxQueue->uxItemSize)) {
        pxQueue->pcWriteTo = pxQueue->pcHead;
    }
}
```

## 临界段问题

```c:queue.c
BaseType_t xQueueGenericSend(..., TickType_t xTicksToWait)
{
    taskENTER_CRITICAL();  // 进入临界段

    for (;;) {
        // ... 检查、拷贝 ...

        if (/* 需要等待 */) {
            taskEXIT_CRITICAL();  // ⚠️ 临时退出临界段
            taskYIELD();          //    让出 CPU
            taskENTER_CRITICAL(); //    重新进入
            // ...
        }
    }

    taskEXIT_CRITICAL();
}
```

**注意**：在临界段中不能调用 `taskYIELD()`，所以需要临时退出临界段再 yield。

## 本节小结

- **xQUEUE** 核心字段：`pcHead`（存储区）、`pcWriteTo`/`pcReadFrom`（读写指针）、`uxMessagesWaiting`（消息数）、两个等待链表
- **循环队列**：写指针/读指针超出末尾时自动回到头部
- **阻塞机制**：队列满/空时任务放入等待链表，调用 `taskYIELD()` 让出 CPU
- **任务唤醒**：取数据时唤醒等待发送的任务，写数据时唤醒等待接收的任务
- `xJustPeeking` 参数：可实现"只查看不取出"（`xQueuePeek()`）
