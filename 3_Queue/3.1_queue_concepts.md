# 3.1 概念：FIFO / 消息传递 / 阻塞机制

## 队列（Queue）是什么

队列是 FreeRTOS 中**任务间通信**的核心机制。数据从一端写入（FIFO），从另一端读取。

```
[IMAGE_PROMPT: Queue FIFO diagram. Show a horizontal box representing queue with head on left and tail on right. Show arrows: data1, data2, data3 being written to tail (enqueue). data1 being read from head (dequeue). Show head and tail pointers. Label "写入（入队）" and "读取（出队）" in Chinese. Show queue items numbered 0, 1, 2. Show "FIFO: 先进先出" label at top.}
```

### 核心特性

| 特性 | 说明 |
|------|------|
| **FIFO** | 先入先出，先发送的数据先被接收 |
| **线程安全** | 写和读可以同时被不同任务调用 |
| **阻塞** | 队列空时读任务阻塞，队列满时写任务阻塞 |
| **拷贝传递** | 数据整体拷贝进队列，不是指针传递 |

## 基本概念

### 队列 vs 消息

队列是一种**消息队列**的实现。发送方把数据拷贝进队列，接收方从队列拷贝出来。

```c
// 发送：整块数据拷贝进队列
uint32_t data = 100;
xQueueSend(xQueue, &data, portMAX_DELAY);  // data 的值被拷贝

// 接收
uint32_t received;
xQueueReceive(xQueue, &received, portMAX_DELAY);  // received 收到 100
```


## 阻塞机制

队列操作可以带阻塞时间（timeout）。

### 读阻塞

```c
// 队列空时，任务进入 Blocked 态，等最多 100ms
BaseType_t result = xQueueReceive(xQueue, &buf, pdMS_TO_TICKS(100));

// result == pdTRUE   成功收到数据
// result == pdFALSE  超时，没收到
```

| 阻塞时间 | 行为 |
|---------|------|
| `0` | 不阻塞，立即返回 |
| `portMAX_DELAY` | 死等，永不超时（需 `INCLUDE_vTaskSuspend`） |
| `pdMS_TO_TICKS(100)` | 等 100ms，超时返回 |

### 写阻塞

```c
// 队列满时，任务进入 Blocked 态，等最多 100ms
BaseType_t result = xQueueSend(xQueue, &data, pdMS_TO_TICKS(100));
```

### 多任务等待同一队列

```
[IMAGE_PROMPT: Multiple tasks waiting on queue diagram. Show queue with 3 tasks blocked on the receive side: Task A, Task B, Task C all waiting to receive. Show Task A label "优先级最高". Show when data arrives, Task A gets it first (unblocked). Label "高优先级任务先被唤醒" in Chinese. Show blocked tasks in a separate waiting area with arrows to queue.}
```

**唤醒顺序**：当数据到达时，**优先级最高**的等待任务先被唤醒。

## 中断安全版本

在中断中不能调用普通队列函数（可能阻塞），必须用 `FromISR` 版本：

```c
// ❌ 中断中不能这样用
void UART_IRQHandler(void) {
    xQueueSendFromISR(xQueue, &data, NULL);  // 这个可以！
}

// ✅ 正确
void UART_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    uint8_t data = UART->DR;
    xQueueSendFromISR(xQueue, &data, &xHigherPriorityTaskWoken);

    // 如果有任务被唤醒且优先级更高，触发上下文切换
    if (xHigherPriorityTaskWoken == pdTRUE) {
        portYIELD_FROM_ISR();
    }
}
```

### FromISR 特点

| 普通版本 | FromISR 版本 |
|---------|------------|
| `xQueueSend()` | `xQueueSendFromISR()` |
| `xQueueReceive()` | `xQueueReceiveFromISR()` |
| timeout = portMAX_DELAY | timeout = 0（不能阻塞） |
| 可能任务切换 | 返回值指示是否需要切换 |

## 常见使用场景

| 场景 | 示例 |
|------|------|
| 传感器数据上报 | 传感器任务采集 → 队列 → 处理任务 |
| 命令传递 | 按键任务检测 → 队列 → 控制任务 |
| 日志输出 | 任务产生日志 → 队列 → 串口任务输出 |
| 事件通知 | 中断产生数据 → FromISR → 任务处理 |

---

## 本节小结

- 队列是 FIFO 结构，数据拷贝进/拷贝出
- `xQueueCreate(队列深度, 元素大小)` 创建队列
- 读/写操作可以带阻塞时间（`portMAX_DELAY` = 死等）
- 中断中必须用 `FromISR` 版本，返回值指示是否需要 `portYIELD_FROM_ISR`
- 多任务等待同一队列时，高优先级任务优先被唤醒
