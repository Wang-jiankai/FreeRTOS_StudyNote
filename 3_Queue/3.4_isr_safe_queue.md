# 3.4 中断安全队列：FromISR 版本

## 为什么需要 FromISR 版本

普通队列操作可能在内部调用 `taskYIELD()`（触发调度），而中断中不能进行任务切换。

```c
// 普通版本可能触发任务切换
xQueueSend(xQueue, &data, portMAX_DELAY);
//     ↑
//     └─ 内部可能调用 taskYIELD()
```

中断中应该用 `FromISR` 版本，它：
- **不阻塞**（timeout 只能是 0）
- **不直接触发任务切换**，而是通过返回值告诉你是否需要切换

## FromISR 源码 trace

### xQueueSendFromISR

```c:queue.c
BaseType_t xQueueSendFromISR(QueueHandle_t xQueue,
                              const void *pvItemToQueue,
                              BaseType_t *pxHigherPriorityTaskWoken)
{
    BaseType_t xReturn = pdFALSE;

    /* 不进入临界段，而是用特殊锁机制 */
    if (xQueue->cIsLocked > (int8_t)0) {
        /* 队列被锁定（中断嵌套中），不能直接操作 */
        xReturn = errQUEUE_FULL;
    } else {
        /* 直接拷贝数据（不阻塞，不 yield） */
        (void)prvCopyDataToQueue(xQueue, pvItemToQueue, queueSEND_TO_BACK);

        /* 唤醒等待接收的任务 */
        if (listCURRENT_LIST_LENGTH(&pxQueue->xTasksWaitingToReceive) > 0) {
            if (xTaskRemoveFromEventList(&pxQueue->xTasksWaitingToReceive) == pdTRUE) {
                /* 标记需要任务切换 */
                *pxHigherPriorityTaskWoken = pdTRUE;
            }
        }
    }

    return xReturn;
}
```

### 关键点：cIsLocked 机制

```c
/* 中断嵌套计数 */
typedef struct QueueDef {
    volatile int8_t cIsLocked;  // 锁计数
    // ...
} xQUEUE;

/* 任务ENTER_CRITICAL时递增，EXIT_CRITICAL时递减 */
/* cIsLocked > 0 时，FromISR 不操作队列，标记 errQUEUE_FULL */
```

## portYIELD_FROM_ISR

```c
void vPortYieldFromISR(void) {
    /* 触发 PendSV，实现任务切换 */
    portYIELD();
}
```

使用时：

```c
void UART_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    xQueueSendFromISR(xQueue, &data, &xHigherPriorityTaskWoken);

    /* ⚠️ 关键：判断是否需要切换任务 */
    if (xHigherPriorityTaskWoken == pdTRUE) {
        /* 唤醒的任务优先级更高，应该立即切换 */
        portYIELD_FROM_ISR();
        /* 等价于：__asm volatile ("svc 0") 或直接触发 PendSV */
    }
}
```

## 常见错误

### 错误 1：忘记判断 pxHigherPriorityTaskWoken

```c
// ❌ 错误：没有判断是否需要切换
void UART_IRQHandler(void) {
    uint8_t ch = USART1->DR;
    xQueueSendFromISR(xQueue, &ch, NULL);  // NULL = 不关心返回值

    // 可能高优先级任务已就绪，但没触发切换
}

// ✅ 正确
void UART_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xQueueSendFromISR(xQueue, &ch, &xHigherPriorityTaskWoken);

    if (xHigherPriorityTaskWoken == pdTRUE) {
        portYIELD_FROM_ISR();
    }
}
```

### 错误 2：在 FromISR 中用非零超时

```c
// ❌ FromISR 不支持超时
xQueueSendFromISR(xQueue, &data, pdMS_TO_TICKS(100));  // ❌ 超时无效
xQueueReceiveFromISR(xQueue, &buf, pdMS_TO_TICKS(10)); // ❌

// ✅ FromISR 第三个参数是 pxHigherPriorityTaskWoken，不是超时
xQueueSendFromISR(xQueue, &data, &xHigherPriorityTaskWoken); // ✅
```

### 错误 3：中断优先级设错

```
FreeRTOS 中断配置关键：

configMAX_SYSCALL_INTERRUPT_PRIORITY = 5

高于或等于此优先级（数值小于5）的中断：
- 不能调用任何 FreeRTOS API
- 不能触发任务切换

低于此优先级（数值大于5）的中断：
- 可以调用 FromISR 系列
- portYIELD_FROM_ISR() 能正常触发切换
```

```c
// NVIC 配置示例
NVIC_SetPriority(UART1_IRQn, 6);  // ✅ 优先级 6 > 5，可以调 FromISR
NVIC_SetPriority(TIM1_UP_IRQn, 4); // ❌ 优先级 4 < 5，不能调 FreeRTOS API
```

## 完整中断处理模板

```c
void USART1_IRQHandler(void) {
    /* 1. 获取中断状态 */
    if (USART1->SR & USART_SR_RXNE) {
        /* 2. 接收数据 */
        uint8_t ch = USART1->DR;

        /* 3. 发送到队列 */
        BaseType_t xHigherPriorityTaskWoken = pdFALSE;
        if (xQueueSendFromISR(xQueue, &ch, &xHigherPriorityTaskWoken) == errQUEUE_FULL) {
            /* 队列满，数据丢失（可以加错误计数） */
            g_queue_full_count++;
        }

        /* 4. 判断是否需要切换任务 */
        portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
    }
}
```

## FromISR 使用检查清单

```
□ 中断优先级是否 < configMAX_SYSCALL_INTERRUPT_PRIORITY？
□ 是否传入了 &pxHigherPriorityTaskWoken 参数（非 NULL）？
□ 是否判断了 xHigherPriorityTaskWoken 并调用 portYIELD_FROM_ISR？
□ FromISR 中是否用了超时？（不应使用）
□ 是否处理了 errQUEUE_FULL 情况？（队列满时数据是否丢弃？）
```

## 本节小结

- `FromISR` 版本不阻塞，返回值指示是否需要任务切换
- `pxHigherPriorityTaskWoken` 参数必须传入非 NULL 值
- 判断为 `pdTRUE` 后必须调用 `portYIELD_FROM_ISR()`
- 中断优先级必须低于 `configMAX_SYSCALL_INTERRUPT_PRIORITY` 才能调 API
- 队列满时数据丢失，需要根据场景决定是否加缓冲或告警
