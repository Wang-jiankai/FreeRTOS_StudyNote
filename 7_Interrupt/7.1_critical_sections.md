# 7.1 临界段：taskENTER_CRITICAL / taskEXIT_CRITICAL

## 什么是临界段

临界段是**不允许中断打扰的代码区**。

```
[IMAGE_PROMPT: Critical section diagram. Timeline showing task execution with two critical sections marked. Inside critical section: no interrupts (hatched/gray). Outside: interrupts allowed. Label "临界段" and "可中断区域". Show that entering critical section disables interrupts, exiting re-enables them. Use Chinese labels.}
```

## API

### 进入和退出

```c
void taskENTER_CRITICAL(void);   // 进入临界段
void taskEXIT_CRITICAL(void);    // 退出临界段
```

### 使用示例

```c
void access_shared_data(void) {
    taskENTER_CRITICAL();  // 关闭中断

    // 操作共享数据（必须快！）
    shared_variable++;

    taskEXIT_CRITICAL();   // 重新开启中断
}
```

---

## 临界段的本质

临界段通过**关闭中断**实现：

```c:portmacro.h
// portmacro.h 中的实现
#define taskENTER_CRITICAL()    vPortEnterCritical()
#define taskEXIT_CRITICAL()     vPortExitCritical()

// vPortEnterCritical() 实际实现（port.c）：
// - 设置 BASEPRI 屏蔽低优先级中断
// - 增加 per-core 嵌套计数 ucCriticalNesting
// - 如果嵌套计数为 1 且在中断中，直接退出（不重复操作）
```

---

## 嵌套

临界段可以嵌套：

```c
taskENTER_CRITICAL();  // 嵌套层数 1
// ...
    taskENTER_CRITICAL();  // 嵌套层数 2
    // ...
    taskEXIT_CRITICAL();   // 嵌套层数 1
// ...
taskEXIT_CRITICAL();    // 嵌套层数 0，打开中断
```

---

## 注意事项

### 1. 临界段必须快

```c
// ❌ 错误：临界段内做耗时操作
void bad_critical(void) {
    taskENTER_CRITICAL();
    vTaskDelay(pdMS_TO_TICKS(100));  // ❌ 不能延时！
    taskEXIT_CRITICAL();
}

// ✅ 正确：快速操作
void good_critical(void) {
    taskENTER_CRITICAL();
    shared_var = calculate();
    taskEXIT_CRITICAL();
}
```

### 2. 临界段内不能调用 FreeRTOS API

```c
// ❌ 错误：临界段内调用可能阻塞的 API
void bad_critical(void) {
    taskENTER_CRITICAL();
    xQueueSend(xQueue, &data, portMAX_DELAY);  // ❌ 可能阻塞！
    taskEXIT_CRITICAL();
}
```

### 3. 临界段嵌套要配对

```c
// ❌ 错误：嵌套不配对
void bad_critical(void) {
    taskENTER_CRITICAL();
    if (condition) {
        taskEXIT_CRITICAL();   // 提前退出
        return;                // 没配对！
    }
    taskEXIT_CRITICAL();
}

// ✅ 正确：用 goto 统一出口
void good_critical(void) {
    taskENTER_CRITICAL();
    if (condition) {
        goto cleanup;  // 跳到统一出口
    }

cleanup:
    taskEXIT_CRITICAL();
}
```

## 与调度器挂起的区别

| 特性 | 临界段 | vTaskSuspendAll |
|------|--------|-----------------|
| 关闭中断 | ✅ | ❌ |
| 禁止任务切换 | ✅ | ✅ |
| 中断中可用 | ✅ | ❌ |
| 使用场景 | 保护共享数据 | 保护数据结构完整性 |

## 中断优先级与临界段

### configMAX_SYSCALL_INTERRUPT_PRIORITY

```c
// FreeRTOSConfig.h
#define configMAX_SYSCALL_INTERRUPT_PRIORITY  5
```

**高于此优先级**（数值 < 5）的中断**不受临界段影响**：

## 本节小结

- **临界段** = 关闭中断 + 禁止调度
- 必须快进快出，不能延时或阻塞
- 可以嵌套，嵌套计数要配对
- 临界段只影响优先级 ≤ `configMAX_SYSCALL_INTERRUPT_PRIORITY` 的中断
