# 2.2 源码解析：vTaskSwitchContext trace

## 调度器何时运行

| 触发时机 | 说明 |
|---------|------|
| **Tick 中断** | 每次 SysTick 中断，检查是否需要切换 |
| **任务主动让出** | `taskYIELD()` → 触发 PendSV |
| **高优先级任务就绪** | 在当前任务或 Tick 中断中检测到 |
| **Port Yield** | 一些移植层特殊机制 |

---

## 核心函数：vTaskSwitchContext

此函数在 `tasks.c` 中，负责**选择下一个要运行的任务**。

```c:tasks.c
void vTaskSwitchContext(void)
{
    if (uxSchedulerSuspended != (UBaseType_t)pdFALSE) {
        /* 调度器被挂起，不切换 */
        xYieldPending = pdTRUE;  // 标记需要切换，等恢复后处理
        return;
    }

    /* 找到最高优先级的就绪任务 */
    taskENTER_CRITICAL();

    /* 从高到低遍历优先级，找非空就绪队列 */
    for (uxTopPriority = uxTopReadyPriority;
         uxTopPriority > 0;
         uxTopPriority--) {
        if (listCURRENT_LIST_LENGTH(
                &(pxReadyTasksLists[uxTopPriority - 1])) > 0)
        {
            /* 找到了，从队列头部取下一个任务 */
            listGET_OWNER_OF_NEXT_ENTRY(
                pxCurrentTCB,
                &(pxReadyTasksLists[uxTopPriority - 1]));
            break;
        }
    }

    taskEXIT_CRITICAL();
}
```

## listGET_OWNER_OF_NEXT_ENTRY 宏

这个宏负责从链表中取出下一个任务：

```c:tasks.c
/* 定义在 tasks.c */
#define listGET_OWNER_OF_NEXT_ENTRY(pxTCB, pxList) \
    do { \
        /* 获取当前节点 */ \
        ListItem_t *pxIndex = pxList->pxIndex; \
        /* 移动到下一个节点 */ \
        if (pxIndex == listGET_END_MARKER(pxList)) { \
            pxIndex = listGET_HEAD(pxList); \
        } else { \
            pxIndex = listGET_NEXT(pxIndex); \
        } \
        /* 获取任务 TCB */ \
        pxTCB = listGET_LIST_ITEM_OWNER(pxIndex); \
    } while (0)
```

**作用**：实现**轮转**（Round-robin），每次调用取链表中的下一个节点，下次再取再下一个，形成轮转。

## 完整的调度时机

### 1. Tick 中断触发调度

```c
void xTaskIncrementTick(void)  // 在 SysTick_Handler 中被调用
{
    BaseType_t switched = pdFALSE;

    /* 检查延时队列，唤醒到期的任务 */
    if (taskIncrementTick()) {  // 返回 pdTRUE = 有任务到期
        switched = pdTRUE;
    }

    /* 如果有任务切换需求 */
    if (switched) {
        /* 触发 PendSV 切换 */
        portYIELD_WITHIN_API();
    }
}
```

### 2. 任务调用 vTaskDelay

```c
void vTaskDelay(TickType_t xTicksToDelay)
{
    // ...
    if (xShouldYield == pdTRUE) {
        taskYIELD();  // 触发 PendSV
    }
}
```

### 3. portYIELD_WITHIN_API

```c:portmacro.h
#define portYIELD_WITHIN_API()  portYIELD()

/* ARM Cortex-M */
#define portYIELD() \
    __asm volatile ( \
        " MOVW R0, #0xEDFF \n"   \
        " MOVT R0, #0xE000 \n"    \
        " LDR R0, [R0] \n"        \
        " STR R0, [R0] \n"         \
    )
/* 触发 PendSV 中断 */
```

## 调度器的禁止与恢复

### vTaskSuspendAll / xTaskResumeAll

某些操作需要临时禁止调度器防止中断：

```c
void vTaskSuspendAll(void) {
    uxSchedulerSuspended++;
}

BaseType_t xTaskResumeAll(void) {
    uxSchedulerSuspended--;

    if (uxSchedulerSuspended == 0) {
        /* 处理挂起期间标记的切换 */
        if (xYieldPending == pdTRUE) {
            xYieldPending = pdFALSE;
            return pdTRUE;  // 需要切换
        }
    }
    return pdFALSE;
}
```

**使用场景**：中断处理函数中，某些操作不能被打断。

## 本节小结

- `vTaskSwitchContext` 核心逻辑：从高到低遍历优先级，找到第一个非空队列，取出下一个任务
- `pxCurrentTCB` 指向当前任务 TCB，切换时更新
- `uxTopReadyPriority` 是优化变量，避免每次都从最高优先级遍历
- `uxSchedulerSuspended` 挂起调度器期间，切换标记延迟执行
- `portYIELD()` 通过触发 PendSV 中断实现真正的 CPU 切换
