# 2.4 源码解析：Cortex-M 上下文切换原理

## 上下文（Context）是什么

上下文 = CPU 寄存器的完整状态。切换任务时，必须保存当前任务的所有寄存器，恢复新任务的所有寄存器。

## 栈帧（Stack Frame）

任务被中断时，CPU 自动压栈部分寄存器，形成"中断栈帧"：

```c
/* ARM 自动压栈的寄存器（硬件） */
__asm void PendSV_Handler(void) {
    /* 硬件自动压栈（中断入口时） */
    /* xPSR, PC, LR, R12, R3, R2, R1, R0 被压入当前 PSP 栈 */
}
```

## 完整的上下文切换步骤

### Step 1：保存当前任务上下文

```c
void PendSV_Handler(void)
{
    __asm volatile (
        /* --- 保存上下文 --- */

        /* 获取当前任务的 PSP（当前栈指针） */
        " MRS     r0, PSP               \n"

        /* 保存 R4-R11（通用寄存器，需要软件保存） */
        " STMDB   r0!, {r4-r11}         \n"
        /* r0 现在指向 R11 所在位置 */

        /* 保存当前任务的栈指针到 TCB */
        " MRS     r1, PRIMASK           \n"
        " CPSID   i                    \n"
        " LDR     r2, =pxCurrentTCB    \n"
        " LDR     r3, [r2]             \n"
        " STR     r0, [r3]             \n"
        " MSR     PRIMASK, r1           \n"
    );
}
```

### Step 2：调用任务选择函数

```c
    __asm volatile (
        /* --- 选择下一个任务 --- */
        " BL      vTaskSwitchContext   \n"
        /* vTaskSwitchContext 更新 pxCurrentTCB */
    );
```

### Step 3：恢复新任务上下文

```c
    __asm volatile (
        /* --- 恢复上下文 --- */

        /* 获取新任务的栈指针 */
        " LDR     r0, =pxCurrentTCB    \n"
        " LDR     r1, [r0]              \n"
        " LDR     r0, [r1]              \n"

        /* 恢复 R4-R11 */
        " LDMIA   r0!, {r4-r11}        \n"

        /* 更新 PSP（栈指针寄存器） */
        " MSR     PSP, r0               \n"

        /* 返回并恢复特权模式（如果需要） */
        " MOVS    r0, #0               \n"
        " MSR     CONTROL, r0           \n"

        /* 退出中断，返回到新任务 */
        " BX      lr                    \n"
    );
}
```

## 任务第一次运行时的特殊性

### 问题

新任务没有"上一次保存的上下文"，第一次运行时不存在"恢复"操作。

### 解决方案

在 `pxPortInitialiseStack` 中模拟一个"中断刚发生"的栈帧：

```c
StackType_t *pxPortInitialiseStack(
    StackType_t *pxTopOfStack,
    TaskFunction_t pxCode,
    void *pvParameters)
{
    /* 模拟中断栈帧：硬件自动压了 xPSR, PC, LR, R12, R3, R2, R1, R0 */
    pxTopOfStack--;
    *pxTopOfStack = portINITIAL_XPSR;        /* xPSR = 0x01000000 */

    pxTopOfStack--;
    *pxTopOfStack = ((StackType_t)pxCode);   /* PC = 任务函数地址 */
    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)prvTaskExitError; /* LR = 退出函数 */

    /* R0-R12 初始值 = 0 */
    for (int i = 12; i >= 0; i--) {
        pxTopOfStack--;
        *pxTopOfStack = 0;
    }

    /* R0 = pvParameters（任务参数） */
    pxTopOfStack--;
    *pxTopOfStack = (StackType_t)pvParameters;  /* R0 */

    return pxTopOfStack;
}
```

### prvTaskExitError

```c
/* 任务函数正常返回时调用这个函数 */
static void prvTaskExitError(void) {
    /* 如果任务函数返回了，执行这里 */
    /* 通常进入死循环或删除任务 */
    for(;;);
}
```

**设计意图**：正常情况下任务函数是 `while(1)` 永不返回。但如果写错了返回了，就会进入 `prvTaskExitError` 防止程序跑飞。

## 本节小结

- **上下文** = CPU 所有寄存器状态，保存在栈上
- **MSP vs PSP**：中断/内核用 MSP，任务用 PSP
- **中断栈帧**：硬件自动保存 xPSR/PC/LR/R12/R3-R0，R4-R11 需软件保存
- **保存**：从 PSP 获取当前栈顶 → 保存 R4-R11 → 更新 TCB
- **恢复**：从 TCB 获取栈顶 → 恢复 R4-R11 → 更新 PSP → 返回新任务
- **首次运行**：用 `pxPortInitialiseStack` 构造模拟栈帧，伪造"中断刚发生"的状态
