# 2.1 概念：抢占式 / 协作式 / 时间片

## 三种调度策略

FreeRTOS 支持三种调度策略，通过 `FreeRTOSConfig.h` 配置：

| 策略 | 配置项 | 说明 |
|------|--------|------|
| **抢占式调度** | `configUSE_PREEMPTION = 1` | 高优先级任务就绪立即抢占 |
| **协作式调度** | `configUSE_PREEMPTION = 0` | 任务主动调用 vTaskYield 才切换 |
| **时间片轮转** | `configUSE_TIME_SLICING = 1` | 同优先级任务轮转执行 |

---

## 抢占式调度（Preemptive）✅ 最常用

### 核心规则

> **高优先级任务就绪 → 立即抢占正在运行的低优先级任务**

```

Task H (high priority) shown in red at top, Task M (medium) in blue, Task L (low) in green. Show timeline: Task L running from 0-3ms, at 3ms Task H becomes ready and preempts Task L immediately, Task H runs until 7ms then blocks, Task L resumes. Label preemption points with Chinese labels "抢占" and arrows. Show task states (Running/Blocked/Ready) as labels above each task bar.]
```

### 特点

- **响应快**：高优先级任务几乎无延迟
- **可抢占低优先级任务的任何代码点**（非函数边界）
- **最常用**，适合大多数实时嵌入式场景

### 代码示例

```c
// 任务 L（低优先级）
void low_task(void *param) {
    while (1) {
        printf("Low task running\r\n");  // 可能执行到一半被打断
        vTaskDelay(pdMS_TO_TICKS(10));
    }
}

// 任务 H（高优先级）
void high_task(void *param) {
    while (1) {
        printf("High task running\r\n");  // 一就绪立即抢
        vTaskDelay(pdMS_TO_TICKS(5));
    }
}
```

## 协作式调度（Cooperative）

### 核心规则

> **任务自己让出 CPU 才切换，不主动抢占**

任务切换的时机：
- `vTaskDelay()` 延时到期
- `vTaskYield()` 主动让出
- 任务阻塞（等待队列、信号量等）

```
[IMAGE_PROMPT: Cooperative scheduling timeline diagram. Same 3 tasks layout as previous but now with labels showing cooperative mode. Show Task L runs from 0-5ms, at 5ms it calls vTaskDelay and blocks, then scheduler switches to Task M. Task H never preempts because it never blocks. Add labels in Chinese "协作模式：任务主动让出才切换". Show only voluntary yield points as transition arrows.]
```

### 特点

- **无随机抢占**，执行顺序可预测
- **响应较慢**：高优先级任务要等低优先级任务主动让出
- **省电**：任务阻塞时 CPU 可深度睡眠
- **适合功耗敏感场景**

### 注意的问题

```c
// ❌ 低优先级任务死循环，永远不让出 CPU
void bad_low_task(void *param) {
    while (1) {
        // 没有 vTaskDelay，没有阻塞
        // 高优先级任务永远无法运行！
        do_critical_work();
    }
}

// ✅ 每次循环至少让出一次
void good_low_task(void *param) {
    while (1) {
        do_critical_work();
        vTaskDelay(pdMS_TO_TICKS(1));  // 让出 CPU
    }
}
```

## 时间片轮转（Time Slicing）

### 核心规则

> **同优先级任务，每个 Tick 切换一次**

- 每个任务运行一个 Tick 的时间片
- 时间片用完，调度器切换到同优先级的下一个任务
- 如果同优先级只有一个任务，时间片无意义

```
[IMAGE_PROMPT: Time slicing timeline for 3 tasks same priority. Horizontal timeline with tick marks at each ms (assuming 1ms tick). Task A, B, C all priority 2. Show at Tick 1: Task A runs, at Tick 2: Task B runs, at Tick 3: Task C runs, at Tick 4: Task A runs again. Label each time slice with task name and tick number. Use different colors for each task. Show "时间片轮转" label at top in Chinese.]
```

### 配置

```c
// FreeRTOSConfig.h
#define configUSE_TIME_SLICING    1   // 1=开启（默认） 0=关闭
```

### 关闭时间片的情况

关闭后，同优先级任务不会自动轮转，只有一个任务会持续运行直到：
- 自己阻塞
- 被更高优先级抢占
- 被 `vTaskYield()` 主动让出

---

## 三种策略组合

| 模式 | PREEMPTION | TIME_SLICING | 效果 |
|------|------------|--------------|------|
| 模式 1 | 1 | 1 | 抢占式 + 时间片（**最常用**） |
| 模式 2 | 1 | 0 | 抢占式，无时间片（同优先级靠阻塞切换） |
| 模式 3 | 0 | 1 | 协作式 + 时间片（同优先级轮转） |
| 模式 4 | 0 | 0 | 纯协作式（只有 vTaskYield 触发切换） |

### 最常用配置：模式 1

```c
#define configUSE_PREEMPTION    1
#define configUSE_TIME_SLICING  1
```

## 空闲任务（Idle Task）

当没有其他就绪任务时，调度器运行空闲任务。

```c
void vApplicationIdleHook(void) {
    // 空闲时执行（不能调用阻塞 API）
    // 常用于进入低功耗模式
    __WFI();  // Wait For Interrupt，CPU 休眠
}
```

**注意**：
- 空闲任务优先级为 0（最低）
- 不能调用会阻塞的函数（`vTaskDelay()` 可以但不推荐）

---

## 本节小结

- **抢占式调度** = 高优先级就绪立即运行，最常用
- **时间片轮转** = 同优先级任务轮流执行一个 Tick
- **协作式调度** = 任务主动让出才切换，功耗低但响应慢
- `configUSE_PREEMPTION = 1` + `configUSE_TIME_SLICING = 1` 是最常用的组合
- 空闲任务在无任务运行时执行，可用于低功耗
