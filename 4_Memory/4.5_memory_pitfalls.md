# 4.5 踩坑实录：内存碎片 / 对齐 / 泄漏

## 坑 1：heap 耗尽导致 hardfault

### 现象

系统随机死机，或 `xTaskCreate` 返回 `errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY`。

### 排查

```c
// 检查 heap 是否耗尽
void check_heap(void) {
    size_t free = xPortGetFreeHeapSize();
    printf("Free heap: %u bytes\r\n", free);
    if (free < 256) {
        printf("WARNING: Heap nearly exhausted!\r\n");
    }
}
```

### 常见原因

```c
// ❌ 原因 1：heap 设太小
#define configTOTAL_HEAP_SIZE ((size_t)4096)  // 不够

// ❌ 原因 2：任务栈设置太大
xTaskCreate(task_func, "Big", 2048, ...);  // 每个 8KB，太费

// ❌ 原因 3：频繁创建删除没释放
while (1) {
    uint8_t *buf = pvPortMalloc(1024);  // 没 free
    // ...
    // buf 指针丢失，内存泄漏
}
```

### 解决

```c
// ✅ 增加 heap 大小
#define configTOTAL_HEAP_SIZE ((size_t)(16 * 1024))  // 16KB

// ✅ 合理设置栈大小
xTaskCreate(task_func, "Normal", 256, ...);  // 1KB 足够

// ✅ 及时释放内存
uint8_t *buf = pvPortMalloc(1024);
if (buf != NULL) {
    // 使用...
    vPortFree(buf);  // 释放
}
```

## 坑 2：内存对齐问题

### 现象

hardfault 或分配返回地址不是 8 的倍数。

### 原因

ARM Cortex-M 要求访问对齐的地址：

```c
// ❌ 手动分配了不对齐的内存
void *ptr = (void *)0x20000001;  // 奇数地址，不对齐
uint32_t val = *(uint32_t *)ptr;  // hardfault！
```

### 解决

```c
// ✅ 使用 pvPortMalloc，它保证 8 字节对齐
void *ptr = pvPortMalloc(100);
uint32_t *pval = (uint32_t *)ptr;  // 对齐的
*pval = 42;  // OK

// ✅ 如果需要强制对齐
#define ALIGN_MASK (portBYTE_ALIGNMENT - 1)
uint8_t *raw = pvPortMalloc(size + ALIGN_MASK);
uint8_t *aligned = (uint8_t *)(((uintptr_t)raw + ALIGN_MASK) & ~ALIGN_MASK);
```

## 坑 3：内存碎片导致分配失败

### 现象

系统运行一段时间后，`pvPortMalloc` 开始返回 NULL，但 `xPortGetFreeHeapSize()` 显示还有大量剩余。

### 原因

碎片化导致没有**连续的**大块可用：

```
碎片化的 heap：
[已用 100][空闲 50][已用 200][空闲 40][已用 100][空闲 30]...
总计空闲：50+40+30 = 120 字节
但最大连续空闲块：只有 50 字节
如果请求 80 字节 → 分配失败！
```

### 场景

```c
// ❌ heap_2 会产生碎片
void leak_task(void *p) {
    uint8_t *buf;
    while (1) {
        buf = pvPortMalloc(100);  // 分配
        vTaskDelay(pdMS_TO_TICKS(100));
        // 没 free！碎片持续累积
    }
}
```

### 解决

```c
// ✅ 用 heap_4（支持合并）
#define configUSE_HEAP_4 1

// ✅ 避免频繁创建/删除
// ✅ 监控最小剩余堆
if (xPortGetMinimumEverFreeHeapSize() < 1024) {
    printf("WARNING: Heap fragmentation detected!\r\n");
}
```

## 坑 4：内存泄漏

### 现象

系统运行越久，可用内存越少，最终耗尽。

### 泄漏场景

```c
// ❌ 场景 1：malloc 不 free
void bad_task(void *p) {
    uint8_t *buf = pvPortMalloc(256);
    if (buf) {
        init_buffer(buf);
        // 正常退出但没释放
    }
    vTaskDelete(NULL);  // 内存泄漏
}

// ❌ 场景 2：任务持有指针，删除后指针失效
void bad_task(void *p) {
    uint8_t *buf = pvPortMalloc(256);
    while (1) {
        if (some_condition) {
            vTaskDelete(NULL);  // buf 泄漏
        }
        vTaskDelay(pdMS_TO_TICKS(10));
    }
}

// ❌ 场景 3：信号量没释放
void bad_task(void *p) {
    xSemaphoreTake(mutex, portMAX_DELAY);
    // 删除任务前没 Give
    vTaskDelete(NULL);  // mutex 泄漏
}
```

### 检测

```c
// 周期性打印剩余堆，观察是否持续下降
void monitor_task(void *p) {
    size_t last_free = xPortGetFreeHeapSize();
    while (1) {
        size_t curr_free = xPortGetFreeHeapSize();
        printf("Free: %u, Min ever: %u\r\n",
               curr_free, xPortGetMinimumEverFreeHeapSize());

        if (curr_free < last_free - 100) {
            printf("ALERT: Possible memory leak detected!\r\n");
        }
        last_free = curr_free;

        vTaskDelay(pdMS_TO_TICKS(5000));
    }
}
```

### 解决

```c
// ✅ 正确：删除任务前清理
void good_task(void *p) {
    uint8_t *buf = pvPortMalloc(256);
    if (buf == NULL) {
        vTaskDelete(NULL);
    }

    // 使用...

    // 清理
    vPortFree(buf);
    vTaskDelete(NULL);  // 先 free 再删除
}
```

## 坑 5：指针越界访问

### 现象

数据随机损坏，系统行为异常。

### 原因

```c
// ❌ 越界写入
uint8_t *buf = pvPortMalloc(10);
for (int i = 0; i < 20; i++) {  // 写越界
    buf[i] = i;
}
```

### 排查

```c
// ✅ 分配时记录边界
#define BUF_SIZE 100
uint8_t *buf = pvPortMalloc(BUF_SIZE);
memset(buf, 0xA5, BUF_SIZE);  // 用固定值填充，方便调试

// 运行后检查是否被破坏
int corrupted = 0;
for (int i = 0; i < BUF_SIZE; i++) {
    if (buf[i] != 0xA5) {
        corrupted++;
    }
}
if (corrupted > 0) {
    printf("Buffer corrupted at %d bytes!\r\n", corrupted);
}
```

## 坑 6：多任务同时访问同一内存

### 现象

数据不一致，或 hardfault。

### 原因

两个任务同时操作同一缓冲区，没有同步：

```c
// ❌ 不安全：两任务同时修改
void task_1(void *p) {
    while (1) {
        shared_buffer.len++;
        shared_buffer.data[shared_buffer.len] = get_value();
    }
}

void task_2(void *p) {
    while (1) {
        if (shared_buffer.len > 0) {
            process(shared_buffer.data, shared_buffer.len);  // 可能读到不一致的数据
        }
    }
}
```

### 解决

```c
// ✅ 用互斥量保护
SemaphoreHandle_t mutex;

void task_1(void *p) {
    while (1) {
        xSemaphoreTake(mutex, portMAX_DELAY);
        shared_buffer.len++;
        shared_buffer.data[shared_buffer.len] = get_value();
        xSemaphoreGive(mutex);
    }
}
```

## 内存调试清单

```
□ heap 是否耗尽？（xPortGetFreeHeapSize）
□ 是否碎片化？（xPortGetMinimumEverFreeHeapSize 持续下降）
□ 是否有泄漏？（free heap 持续下降）
□ 指针是否对齐？（pvPortMalloc 保证对齐）
□ 是否有越界访问？（边界检查）
□ 多任务是否对同一内存做了同步保护？
□ 删除任务前是否释放了持有的资源？
```

---

## 本节小结

- **heap 耗尽**：增加 `configTOTAL_HEAP_SIZE` 或减小栈大小
- **对齐问题**：只用 `pvPortMalloc`，不用裸指针
- **碎片**：用 heap_4，注意不要频繁分配/释放
- **泄漏**：每个 `pvPortMalloc` 都要有对应的 `vPortFree`
- **越界**：写入前检查边界
- **同步**：多任务访问共享内存要用互斥量保护
