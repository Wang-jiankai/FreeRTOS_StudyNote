# 4.1 概念：五种 heap 策略对比表

## FreeRTOS 内存分配概述

FreeRTOS 的动态内存分配有 **5 种实现**，位于 `FreeRTOS/Source/portable/MemMang/`：

| 文件 | 策略 | 碎片支持 | 确定性 |
|------|------|---------|--------|
| `heap_1.c` | 最简单分配 | ❌ | ✅ 确定性 |
| `heap_2.c` | 最佳匹配 | ❌ | ⚠️ 非确定性 |
| `heap_3.c` | C 库 malloc/free | ✅ | ⚠️ 非确定性 |
| `heap_4.c` | 首次适配 + 合并 | ✅ | ⚠️ 非确定性 |
| `heap_5.c` | heap_4 + 多内存区 | ✅ | ⚠️ 非确定性 |

## heap_1 — 顺序分配（最简单）

### 原理

分配时从低地址向高地址顺序分配，不支持释放。

```
[IMAGE_PROMPT: heap_1 diagram. Horizontal memory block. Show allocation sequence: first malloc(100) takes leftmost 100 bytes, second malloc(50) takes next 50 bytes right after. Label "只分配，不释放" and "简单，但只能一次性使用". Show end marker at far right labeled "configTOTAL_HEAP_SIZE".]
```

### 特点

- ✅ 最简单，代码量最小
- ✅ 确定性：分配时间固定
- ❌ 不能释放内存
- ❌ 内存利用率低

### 适用场景

适合**任务创建后不删除**的场景，如：
- 任务/队列/信号量创建后不删除
- 不需要动态创建/销毁

---

## heap_2 — 最佳匹配（支持释放）

### 原理

释放时标记为可用，分配时找**最接近需求**的空闲块。

```c
// 示例
heap[ ] = [空闲:500] [已用:100] [空闲:200]
// 分配 150 字节
// 找最接近的空闲块：200（而不是500）
// 结果：[已用:100] [已用:150] [剩余:50] [空闲:200]
```

### 特点

- ✅ 支持释放
- ❌ 碎片化：多次分配/释放后会产生很多小碎片
- ⚠️ 非确定性：分配时间取决于空闲块数量
- ❌ 不合并相邻空闲块

### 适用场景

适合**分配大小固定**的场景，如：
- 多个同等大小的任务栈
- 不常创建/删除任务

---

## heap_3 — 标准 C 库

### 原理

直接调用 `malloc()` 和 `free()`，依赖 C 库实现。

```c
// 实际上就是简单封装
void *pvPortMalloc(size_t xSize) {
    return malloc(xSize);
}

void vPortFree(void *pv) {
    free(pv);
}
```

### 特点

- ✅ 代码简单
- ✅ 支持碎片合并（C 库实现）
- ⚠️ 非确定性（C 库决定）
- ⚠️ 线程不安全（需要信号量保护）
- ⚠️ C 库可能很大（嵌入式不友好）

### 适用场景

适合对确定性要求不高的场景，如：
- 大型 MCU，内存充足
- 已有成熟 C 库

---

## heap_4 — 首次适配 + 合并（最常用）✅

### 原理

1. **首次适配**：找到第一个够大的空闲块就分配
2. **相邻合并**：释放时合并相邻的空闲块

### 特点

- ✅ 支持释放
- ✅ 支持碎片合并
- ⚠️ 非确定性
- ✅ 内存利用率较高
- ✅ **最常用**

### 适用场景

几乎所有场景都推荐使用 `heap_4.c`。

---

## heap_5 — heap_4 + 多内存区

### 原理

与 heap_4 相同，但支持**多个不连续的内存区域**：

```c
// 定义多个内存区
static const HeapRegion_t xHeapRegions[] = {
    { (uint8_t *)0x10000000, 0x10000 },  // 外部 SRAM 64KB
    { (uint8_t *)0x20000000, 0x08000 },  // 内部 SRAM 32KB
    { NULL, 0 }  // 结束标记
};

vPortDefineHeapRegions(xHeapRegions);
```

### 特点

- ✅ 支持多块不连续内存
- ✅ 支持碎片合并
- ⚠️ 非确定性
- ⚠️ 配置复杂

### 适用场景

适合有**多块不连续内存**的芯片，如：
- 外部 + 内部 SRAM
- 多块 SRAM 的 MCU

## 对比表

| 特性 | heap_1 | heap_2 | heap_3 | heap_4 | heap_5 |
|------|--------|--------|--------|--------|--------|
| 支持释放 | ❌ | ✅ | ✅ | ✅ | ✅ |
| 碎片合并 | ❌ | ❌ | ✅ | ✅ | ✅ |
| 确定性 | ✅ | ❌ | ❌ | ❌ | ❌ |
| 代码大小 | 最小 | 小 | 依赖库 | 中 | 中 |
| 多内存区 | ❌ | ❌ | ❌ | ❌ | ✅ |
| 适用场景 | 静态系统 | 大小固定 | 内存充足 | **推荐** | 多块内存 |

---

## 本节小结

- **heap_1**：不能释放，只适合简单静态系统
- **heap_2**：支持释放但有碎片，适合分配大小固定的场景
- **heap_3**：用标准 C 库，依赖库实现
- **heap_4**：**最常用**，首次适配 + 合并
- **heap_5**：支持多块不连续内存，适合特殊芯片
