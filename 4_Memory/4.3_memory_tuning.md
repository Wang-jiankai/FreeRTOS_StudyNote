# 4.3 调参指南：heap 大小怎么定

## configTOTAL_HEAP_SIZE

### 基本公式

```
configTOTAL_HEAP_SIZE ≥
    所有任务栈总和
  + 任务 TCB（每个约 96 字节）
  + 队列内存（深度 × 元素大小 × 数量）
  + 信号量/互斥量（少量）
  + heap 自身管理开销（每块 8 字节）
  + 安全余量（20%~30%）
```

### 实际计算示例

```c
// 假设系统配置：
// - 3 个任务，栈分别为 256/512/256 字
// - 2 个队列，深度 10，元素大小 4 字节
// - 2 个信号量

// Step 1: 任务栈
//  ARM Cortex-M: 1 word = 4 bytes
//  任务1: 256 × 4 = 1024 字节
//  任务2: 512 × 4 = 2048 字节
//  任务3: 256 × 4 = 1024 字节
//  小计：4096 字节

// Step 2: TCB
//  每个 TCB 约 96 字节
//  3 × 96 = 288 字节

// Step 3: 队列
//  队列1: 10 × 4 × 2(双向) = 80 字节 × 2 = 160 字节
//  队列2: 10 × 4 = 40 字节
//  小计：200 字节

// Step 4: 信号量
//  每个信号量约 56 字节
//  2 × 56 = 112 字节

// Step 5: heap 管理开销
//  约 8 字节 × (任务数 + 队列数 + 信号量数)
//  8 × 7 = 56 字节

// 总计：4096 + 288 + 200 + 112 + 56 = 4752 字节
// 加 30% 安全余量：4752 × 1.3 ≈ 6182 字节
// 取整：configTOTAL_HEAP_SIZE = 8KB = 8192 字节
```

## 运行时监控

### 获取剩余堆大小

```c
// 剩余可用堆
size_t xFreeHeapSize = xPortGetFreeHeapSize();
// xPortGetFreeHeapSize() = configTOTAL_HEAP_SIZE - 已分配

printf("Free heap: %u bytes\r\n", xFreeHeapSize);
```

### 获取最小剩余堆（水线）

```c
// 历史最低堆剩余
size_t xMinimumEverFreeBytes = xPortGetMinimumEverFreeHeapSize();

// 如果接近 0，说明 heap 接近耗尽
if (xMinimumEverFreeBytes < 512) {
    printf("WARNING: Heap nearly exhausted!\r\n");
}
```

### 运行时周期性检查

```c
void monitor_task(void *param) {
    while (1) {
        size_t free_heap = xPortGetFreeHeapSize();
        size_t min_free = xPortGetMinimumEverFreeHeapSize();

        printf("Free heap: %u bytes, Min ever: %u bytes\r\n",
               free_heap, min_free);

        if (free_heap < 1024) {
            // 告警：剩余不足 1KB
            trigger_alarm();
        }

        vTaskDelay(pdMS_TO_TICKS(5000));
    }
}
```

## 常见配置参考值

| 系统规模 | 任务数 | 典型 heap 大小 |
|---------|--------|---------------|
| 极简系统 | 1~2 | 4KB |
| 简单系统 | 3~5 | 8KB |
| 中等系统 | 5~10 | 16KB |
| 复杂系统 | 10~20 | 32KB+ |
| 高级系统 | 20+ | 64KB+ |

## 调参中的常见错误

### 错误 1：heap 设太小导致系统崩溃

```c
// ❌ heap 太小
#define configTOTAL_HEAP_SIZE ((size_t)4096)  // 4KB 可能不够

// 现象：xTaskCreate 返回 errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY
// 或者 hardfault
```

### 错误 2：局部变量算进 heap

```c
// ❌ 误以为 heap 用来存大数组
void my_task(void *param) {
    uint8_t buffer[4096];  // 这个在任务栈上，不在 heap 上
    // ...
}

// ✅ 大缓冲区应该动态分配（算 heap）
void my_task(void *param) {
    uint8_t *buffer = pvPortMalloc(4096);  // 算 heap
    if (buffer == NULL) { /* 处理错误 */ }
}
```

### 错误 3：没考虑 heap 碎片

```c
// 场景：反复创建/删除任务
while (1) {
    xTaskCreate(task_func, "Temp", 512, NULL, 1, NULL);
    vTaskDelay(pdMS_TO_TICKS(1000));
    vTaskDelete(task_func);  // 释放 512 字栈
    // heap_4 会合并，但频繁分配/释放可能导致碎片
}
```

## 内存泄漏检测

### 内存泄漏的标志

```c
// 如果 free heap 持续下降，说明有泄漏
while (1) {
    printf("Free heap: %u bytes\r\n", xPortGetFreeHeapSize());
    vTaskDelay(pdMS_TO_TICKS(1000));
}

// 观察：如果每次打印都比上次少，且不回升，说明有泄漏
// Free heap: 8192
// Free heap: 8000
// Free heap: 7816  ← 持续下降
// Free heap: 7632
```

### 常见泄漏场景

```c
// ❌ 泄漏 1：没释放 malloc
void bad_task(void *p) {
    uint8_t *buf = pvPortMalloc(256);
    // 处理...
    // 没有 vPortFree(buf);
}

// ❌ 泄漏 2：任务删除前没释放信号量
void bad_task(void *p) {
    xSemaphoreTake(mutex, portMAX_DELAY);
    // 任务删除时 mutex 没释放
    vTaskDelete(NULL);
}

// ✅ 正确：用完即释放，或在 cleanup 中释放
void good_task(void *p) {
    uint8_t *buf = pvPortMalloc(256);
    // 处理...
    vPortFree(buf);

    // 或用静态分配，不用 heap
    static uint8_t static_buf[256];
}
```

## 调参检查清单

```
□ configTOTAL_HEAP_SIZE 够所有任务栈 + TCB + 队列 + 信号量？
□ 加了 20%~30% 安全余量？
□ heap 设为 4 字节对齐？（ARM 要求 8 字节）
□ 运行时监控 xPortGetFreeHeapSize()？
□ 周期性检查 xPortGetMinimumEverFreeHeapSize()？
□ 任务栈用 xTaskCreate 的 ulStackDepth 参数设置，不依赖 heap？
□ 大缓冲区用静态分配还是动态分配？（权衡）
□ 确认没有内存泄漏？（free heap 持续下降？）
```

---

## 本节小结

- **heap 大小** = 任务栈总和 + TCB + 队列 + 信号量 + 管理开销 + 30% 余量
- **监控**：用 `xPortGetFreeHeapSize()` 和 `xPortGetMinimumEverFreeHeapSize()` 观察
- **泄漏检测**：free heap 持续下降说明有泄漏
- **安全余量**：最少保留 20%~30%，防止运行时动态分配失败
