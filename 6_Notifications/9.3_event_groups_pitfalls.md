# 9.3 踩坑实录：事件标志组

## 坑 1：位被意外清除

**现象**：`xEventGroupWaitBits` 返回值中，期望的位已经不在了。

```c
void task(void *p) {
    EventBits_t bits = xEventGroupWaitBits(
        xEventGroup, BIT_0 | BIT_1,
        pdFALSE,  // 不自动清除
        pdFALSE,
        portMAX_DELAY);

    // ❌ 假设 BIT_0 还在，但实际上可能已被其他任务清除
    if (bits & BIT_0) {
        process_temp();
    }
}
```

**原因**：其他任务在等待期间调用了 `xEventGroupClearBits`。

**解决**：用 `xClearOnExit = pdTRUE` 或者读取返回值后立即判断：

```c
void task(void *p) {
    EventBits_t bits = xEventGroupWaitBits(
        xEventGroup, BIT_0 | BIT_1,
        pdTRUE,  // ✅ 退出时自动清除
        pdFALSE,
        portMAX_DELAY);

    if (bits & BIT_0) {
        process_temp();
    }
}
```

## 坑 2：位值被其他任务覆盖

**现象**：多个任务都在置位，但等待任务发现数据丢失了。

```c
void sensor_task(void *p) {
    while (1) {
        // ❌ 只置一个位，但之前其他位可能已经有值了
        xEventGroupSetBits(xEventGroup, BIT_0);  // 只传温度
    }
}

void process_task(void *p) {
    while (1) {
        EventBits_t bits = xEventGroupWaitBits(
            xEventGroup, BIT_0 | BIT_1, pdTRUE, pdTRUE, portMAX_DELAY);
        // 以为两个都有值，实际上 BIT_1 可能一直是旧的
    }
}
```

**原因**：事件组只传递"事件发生"的信号，不传递数据。温度/湿度的新值需要通过其他方式（队列）传递。

**解决**：事件组只用于同步，数据通过队列传递：

```c
void sensor_task(void *p) {
    while (1) {
        float temp = read_temp();
        xQueueSend(xTempQueue, &temp, portMAX_DELAY);  // 数据通过队列传
        xEventGroupSetBits(xEventGroup, BIT_0);          // 事件通过标志组通知
    }
}
```

## 坑 3：32 位以下平台位操作问题

**现象**：在 16 位 MCU 上，`EventBits_t` 是 16 位，定义超过 16 个位会出问题。

```c
// 16 位 MCU 上
typedef uint16_t EventBits_t;

#define BIT_16 (1UL << 16)  // ❌ 左移 16 位超出范围，结果是 0
```

**解决**：

```c
// ✅ 使用位运算而非直接移位
#define MY_EVENT (EventBits_t)(1 << 0)

// ✅ 或者用统一的事件定义文件
#define EVT_SENSOR_TEMP  0x0001
#define EVT_SENSOR_HUMI  0x0002
#define EVT_BUTTON       0x0004
```

## 坑 4：中断中错误使用 xEventGroupSetBits

**现象**：在中断中调用 `xEventGroupSetBits`，但中断优先级太高，无法触发任务切换。

```c
// ❌ xEventGroupSetBits 不是 FromISR 版本！
void HighPriISR(void) {
    xEventGroupSetBits(xEventGroup, BIT_0);  // ❌ 不应该在 ISR 中用
}

// ✅ 正确：用 xEventGroupSetBitsFromISR
void HighPriISR(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xEventGroupSetBitsFromISR(xEventGroup, BIT_0, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

## 坑 5：等待超时后没有判断返回值

**现象**：等待超时返回 0，但代码假设成功，继续使用无效数据。

```c
void task(void *p) {
    EventBits_t bits = xEventGroupWaitBits(
        xEventGroup, BIT_0 | BIT_1, pdTRUE, pdTRUE, pdMS_TO_TICKS(100));

    // ❌ 超时时 bits == 0，没判断
    if (bits & BIT_0) {
        process_temp();  // 数据可能没更新
    }
}
```

**解决**：始终检查返回值：

```c
void task(void *p) {
    EventBits_t bits = xEventGroupWaitBits(
        xEventGroup, BIT_0 | BIT_1, pdTRUE, pdTRUE, pdMS_TO_TICKS(100));

    if (bits == 0) {
        // 超时了，处理超时逻辑
        handle_timeout();
    } else {
        // bits 中只有满足条件的位
        if (bits & BIT_0) process_temp();
        if (bits & BIT_1) process_humidity();
    }
}
```

## 本节小结

| 坑 | 原因 | 解决 |
|----|------|------|
| 位被意外清除 | 其他任务清除了位 | 用 `xClearOnExit = pdTRUE` 或加锁 |
| 数据丢失 | 事件组只传信号不传数据 | 数据走队列，事件走标志组 |
| 位移位溢出 | 16 位平台超过位数 | 用 `(EventBits_t)(1 << n)` 强转 |
| 中断用错 API | xEventGroupSetBits 非 FromISR | 用 `xEventGroupSetBitsFromISR` |
| 超时不判断 | 没检查返回值是否为 0 | 始终判断返回值 |
